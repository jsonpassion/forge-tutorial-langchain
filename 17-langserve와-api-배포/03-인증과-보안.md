# 인증과 보안

> LangServe API를 안전하게 보호하는 인증·보안 전략을 체계적으로 학습합니다

## 개요

이 섹션에서는 LangServe로 배포한 API에 인증과 보안을 적용하는 방법을 다룹니다. API 키 인증부터 JWT 토큰 인증, 요청별 설정 변경(`per_req_config_modifier`), 입력 검증, 속도 제한까지 — 프로덕션 환경에서 반드시 갖춰야 할 보안 계층을 하나씩 쌓아갑니다.

**선수 지식**: [17.1 LangServe 기초](./01-langserve-기초.md)에서 배운 `add_routes`와 기본 서버 구성, [17.2 FastAPI 통합과 서버 구성](./02-fastapi-통합과-서버-구성.md)에서 배운 미들웨어와 CORS 설정, `per_req_config_modifier` 기본 개념

**학습 목표**:
- API 키 기반의 간단한 인증을 구현할 수 있다
- JWT 토큰을 발급하고 검증하는 인증 시스템을 구축할 수 있다
- `per_req_config_modifier`로 사용자별 권한을 체인에 주입할 수 있다
- Pydantic을 활용한 입력 검증으로 악의적 요청을 차단할 수 있다

## 왜 알아야 할까?

여러분이 멋진 RAG 체인을 만들어 LangServe로 API를 배포했다고 상상해 보세요. 그런데 누구나 이 API에 접근할 수 있다면 어떻게 될까요? 악의적인 사용자가 대량의 요청을 보내 LLM 비용을 폭증시키거나, 다른 사용자의 문서를 열람하거나, 프롬프트 인젝션으로 시스템을 오용할 수 있습니다.

LLM API는 **호출당 비용이 발생**하기 때문에, 전통적인 웹 API보다 보안이 더 중요합니다. GPT-4o 한 번 호출에 수십 원이 들 수 있고, 인증 없이 노출된 엔드포인트는 하루 만에 수백만 원의 비용을 발생시킬 수 있거든요.

이번 세션에서는 문을 잠그는 것(인증)부터, 열쇠를 가진 사람에게만 적절한 방을 안내하는 것(인가), 그리고 들어오는 짐을 검사하는 것(입력 검증)까지 — API 보안의 핵심 3요소를 모두 다룹니다.

## 핵심 개념

### 개념 1: API 키 인증 — 가장 단순한 문잠금

> 💡 **비유**: API 키 인증은 아파트 출입문의 **비밀번호**와 같습니다. 번호를 아는 사람만 건물에 들어올 수 있죠. 간단하지만, 번호가 유출되면 누구나 들어올 수 있다는 한계가 있습니다.

API 키 인증은 가장 기본적인 인증 방식입니다. 클라이언트가 요청 헤더에 미리 발급된 키를 포함하면, 서버가 이를 검증하는 방식이죠. LangServe는 FastAPI의 의존성 주입(Dependency Injection) 시스템을 그대로 활용하므로, `add_routes`의 `dependencies` 매개변수로 인증 로직을 간단하게 적용할 수 있습니다.

```python
# api_key_auth.py — API 키 인증 기본 예제
from fastapi import Depends, FastAPI, Header, HTTPException
from langchain_core.runnables import RunnableLambda
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from typing_extensions import Annotated

from langserve import add_routes

# 유효한 API 키 목록 (실무에서는 DB나 환경 변수로 관리)
VALID_API_KEYS = {"sk-coursekit-abc123", "sk-coursekit-def456"}


async def verify_api_key(x_api_key: Annotated[str, Header()]) -> None:
    """X-API-Key 헤더를 검증합니다."""
    if x_api_key not in VALID_API_KEYS:
        raise HTTPException(
            status_code=401,
            detail="Invalid API key"
        )


app = FastAPI(title="Secured LangServe API")

# 간단한 번역 체인
chain = (
    ChatPromptTemplate.from_messages([
        ("system", "You are a translator. Translate to {language}."),
        ("human", "{text}")
    ])
    | ChatOpenAI(model="gpt-4o", temperature=0)
    | StrOutputParser()
)

# dependencies로 모든 라우트에 인증 적용
add_routes(
    app,
    chain,
    path="/translate",
    dependencies=[Depends(verify_api_key)],  # 핵심: 의존성으로 인증 주입
)

if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000)
```

클라이언트에서 호출할 때는 헤더에 API 키를 포함합니다:

```python
# 클라이언트 호출 예시
import requests

response = requests.post(
    "http://localhost:8000/translate/invoke",
    json={"input": {"text": "안녕하세요", "language": "English"}},
    headers={"X-API-Key": "sk-coursekit-abc123"}  # 인증 헤더
)
print(response.json())
# 출력: {"output": "Hello"}
```

`dependencies` 매개변수의 장점은 OpenAPI 문서(`/docs`)에 인증 요구사항이 자동으로 반영된다는 것입니다. 하지만 이 방식만으로는 **사용자별 로직**(예: 사용자마다 다른 문서에 접근)을 구현하기 어렵습니다. 그것은 뒤에서 배울 `per_req_config_modifier`의 영역이죠.

### 개념 2: JWT 토큰 인증 — 신분증을 가진 사람만 입장

> 💡 **비유**: JWT는 놀이공원의 **자유이용권 팔찌**와 같습니다. 입구에서 신분증을 보여주고(로그인) 팔찌를 받으면(토큰 발급), 이후에는 각 놀이기구(API 엔드포인트)에서 팔찌만 보여주면 됩니다. 팔찌에는 이름, 등급, 유효시간이 적혀 있어서, 매번 신분증을 꺼내지 않아도 되죠.

API 키는 단순하지만, "누가" 요청했는지 구분하기 어렵습니다. JWT(JSON Web Token)는 사용자 정보를 토큰 자체에 담아 전달하므로, 사용자 식별과 권한 관리가 가능합니다.

JWT의 구조는 세 부분으로 이루어져 있습니다:

| 부분 | 역할 | 예시 |
|------|------|------|
| **Header** | 알고리즘, 토큰 타입 | `{"alg": "HS256", "typ": "JWT"}` |
| **Payload** | 사용자 정보(클레임) | `{"sub": "alice", "role": "admin", "exp": 1717200000}` |
| **Signature** | 위변조 방지 서명 | `HMACSHA256(header + payload, secret)` |

```python
# jwt_auth.py — JWT 기반 인증 시스템
import os
from datetime import datetime, timedelta, timezone
from typing import Optional

from fastapi import Depends, FastAPI, HTTPException, status
from fastapi.security import OAuth2PasswordBearer, OAuth2PasswordRequestForm
from jose import JWTError, jwt
from passlib.context import CryptContext
from pydantic import BaseModel
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI
from typing_extensions import Annotated

from langserve import add_routes

# 비밀 키 (실무에서는 반드시 환경 변수로 관리!)
SECRET_KEY = os.getenv("JWT_SECRET_KEY", "your-secret-key-change-in-production")
ALGORITHM = "HS256"
ACCESS_TOKEN_EXPIRE_MINUTES = 30

# 비밀번호 해싱 설정
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

# OAuth2 스키마 — /token 엔드포인트에서 토큰 발급
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="token")


# --- 데이터 모델 ---
class Token(BaseModel):
    access_token: str
    token_type: str


class User(BaseModel):
    username: str
    email: Optional[str] = None
    role: str = "user"
    disabled: bool = False


class UserInDB(User):
    hashed_password: str


# 가상 사용자 DB (실무에서는 실제 DB 사용)
USERS_DB = {
    "alice": UserInDB(
        username="alice",
        email="alice@example.com",
        role="admin",
        disabled=False,
        hashed_password=pwd_context.hash("alice-password"),
    ),
    "bob": UserInDB(
        username="bob",
        email="bob@example.com",
        role="user",
        disabled=False,
        hashed_password=pwd_context.hash("bob-password"),
    ),
}


# --- 인증 유틸리티 함수 ---
def verify_password(plain_password: str, hashed_password: str) -> bool:
    """평문 비밀번호와 해시를 비교합니다."""
    return pwd_context.verify(plain_password, hashed_password)


def authenticate_user(username: str, password: str) -> Optional[UserInDB]:
    """사용자 인증: 아이디와 비밀번호를 검증합니다."""
    user = USERS_DB.get(username)
    if not user or not verify_password(password, user.hashed_password):
        return None
    return user


def create_access_token(data: dict, expires_delta: Optional[timedelta] = None) -> str:
    """JWT 액세스 토큰을 생성합니다."""
    to_encode = data.copy()
    expire = datetime.now(timezone.utc) + (expires_delta or timedelta(minutes=15))
    to_encode.update({"exp": expire})
    return jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)


async def get_current_user(
    token: Annotated[str, Depends(oauth2_scheme)]
) -> User:
    """토큰에서 현재 사용자를 추출합니다."""
    credentials_exception = HTTPException(
        status_code=status.HTTP_401_UNAUTHORIZED,
        detail="Could not validate credentials",
        headers={"WWW-Authenticate": "Bearer"},
    )
    try:
        # 토큰 디코딩
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        username: str = payload.get("sub")
        if username is None:
            raise credentials_exception
    except JWTError:
        raise credentials_exception

    user = USERS_DB.get(username)
    if user is None or user.disabled:
        raise credentials_exception
    return User(**user.model_dump())


# --- FastAPI 앱 설정 ---
app = FastAPI(title="JWT-Secured LangServe API")


@app.post("/token", response_model=Token)
async def login(form_data: Annotated[OAuth2PasswordRequestForm, Depends()]):
    """로그인하여 JWT 토큰을 발급받습니다."""
    user = authenticate_user(form_data.username, form_data.password)
    if not user:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="Incorrect username or password",
            headers={"WWW-Authenticate": "Bearer"},
        )
    access_token = create_access_token(
        data={"sub": user.username, "role": user.role},
        expires_delta=timedelta(minutes=ACCESS_TOKEN_EXPIRE_MINUTES),
    )
    return Token(access_token=access_token, token_type="bearer")


# 체인 정의
chain = (
    ChatPromptTemplate.from_messages([
        ("system", "You are a helpful assistant."),
        ("human", "{question}")
    ])
    | ChatOpenAI(model="gpt-4o", temperature=0.7)
    | StrOutputParser()
)

# JWT 인증이 적용된 라우트
add_routes(
    app,
    chain,
    path="/chat",
    dependencies=[Depends(get_current_user)],  # JWT 검증 의존성
)

if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000)
```

```python
# JWT 인증 클라이언트 사용 흐름

BASE_URL = "http://localhost:8000"

# 1단계: 로그인하여 토큰 발급

)
token = login_response.json()["access_token"]

# 출력: 발급된 토큰: eyJhbGciOiJIUzI1NiIs...

# 2단계: 토큰으로 API 호출

    f"{BASE_URL}/chat/invoke",

    headers={"Authorization": f"Bearer {token}"}  # Bearer 토큰
)
print(response.json()["output"])
# 출력: LangChain은 LLM 기반 애플리케이션 개발을 위한 프레임워크입니다...

bad_response = requests.post(
    f"{BASE_URL}/chat/invoke",
    json={"input": {"question": "비인증 요청"}}
)
print(bad_response.status_code)  # 401
print(bad_response.json())       # {"detail": "Not authenticated"}
```

### 개념 3: per_req_config_modifier — 사용자별 맞춤 설정 주입

> 💡 **비유**: `per_req_config_modifier`는 호텔의 **키카드 시스템**과 같습니다. 모든 손님이 같은 호텔에 묵지만, 키카드에 따라 접근할 수 있는 층과 방이 다릅니다. 마찬가지로, 같은 체인을 호출하더라도 인증된 사용자에 따라 다른 데이터나 설정을 주입할 수 있습니다.

앞서 `dependencies`로 "인증 여부"를 확인했다면, `per_req_config_modifier`는 한 걸음 더 나아가 **인증된 사용자 정보를 체인 내부로 전달**합니다. 이것이 왜 중요할까요? 멀티테넌트(multi-tenant) 환경에서 사용자마다 다른 문서를 검색하거나, 역할에 따라 다른 모델을 사용해야 할 때 필수적이거든요.

```python
# per_req_auth.py — 사용자별 설정 주입 예제
from typing import Dict

from fastapi import FastAPI, HTTPException, Request, status
from fastapi.security import OAuth2PasswordBearer
from jose import JWTError, jwt
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import ConfigurableField, RunnableConfig
from langchain_openai import ChatOpenAI

from langserve import add_routes

SECRET_KEY = "your-secret-key"
ALGORITHM = "HS256"
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="token")

app = FastAPI()

# 역할별 시스템 프롬프트 매핑
ROLE_PROMPTS = {
    "admin": "You are an admin assistant with full access. Be thorough.",
    "user": "You are a helpful assistant. Keep answers concise.",
    "guest": "You are a limited assistant. Only answer general questions.",
}


async def per_req_config_modifier(
    config: Dict, request: Request
) -> Dict:
    """요청마다 사용자 정보를 config에 주입합니다."""
    # Authorization 헤더에서 토큰 추출
    auth_header = request.headers.get("Authorization")
    if not auth_header or not auth_header.startswith("Bearer "):
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="Missing or invalid Authorization header",
        )

    token = auth_header.split(" ")[1]

    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        username = payload.get("sub")
        role = payload.get("role", "guest")
    except JWTError:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="Invalid token",
        )

    # configurable 필드에 사용자 정보 주입
    config["configurable"] = {
        "user_id": username,
        "role": role,
        "system_prompt": ROLE_PROMPTS.get(role, ROLE_PROMPTS["guest"]),
    }
    return config


# ConfigurableField로 런타임 설정 가능한 체인 구성
prompt = ChatPromptTemplate.from_messages([
    ("system", "{system_prompt}"),
    ("human", "{question}")
]).configurable_fields(
    # system_prompt을 configurable로 설정
    messages=ConfigurableField(
        id="system_prompt",
        name="System Prompt",
        description="Role-based system prompt",
    )
)

chain = (
    ChatOpenAI(model="gpt-4o", temperature=0.7)
    | StrOutputParser()
)

# per_req_config_modifier 적용
add_routes(
    app,
    chain,
    path="/chat",
    per_req_config_modifier=per_req_config_modifier,
)
```

핵심 포인트는 토큰에서 추출한 사용자 정보가 **절대 프롬프트에 직접 들어가지 않는다**는 것입니다. `config["configurable"]`에 안전하게 담겨 체인의 런타임 설정으로만 사용되므로, LLM이 이 정보를 누출할 위험이 없습니다.

### 개념 4: CORS 보안 강화 — 인증과 함께 고려할 점

[17.2 FastAPI 통합과 서버 구성](./02-fastapi-통합과-서버-구성.md)에서 `CORSMiddleware` 설정과 `allow_origins`, `allow_methods`, `allow_headers` 등의 구성 방법을 자세히 다뤘습니다. 여기서는 반복하지 않고, **인증 시스템과 결합할 때 놓치기 쉬운 보안 포인트**만 짚어보겠습니다.

첫째, **`allow_credentials=True`와 `allow_origins=["*"]`는 동시에 사용할 수 없습니다**. 브라우저가 이 조합을 거부하거든요. JWT의 `Authorization` 헤더나 쿠키를 사용하는 인증 시스템에서는 `allow_credentials=True`가 필요하므로, 반드시 출처를 명시적으로 지정해야 합니다.

둘째, LangServe는 `invoke`, `stream`, `batch` 모두 **POST** 메서드만 사용합니다. `allow_methods`를 `["POST"]`로 제한하면 불필요한 HTTP 메서드를 통한 공격 표면을 줄일 수 있죠.

셋째, `allow_headers`에는 인증에 사용하는 헤더만 명시적으로 열어두세요. API 키 방식이라면 `"X-API-Key"`를, JWT 방식이라면 `"Authorization"`을 추가하면 됩니다. `["*"]`로 모든 헤더를 허용하면 예상치 못한 헤더를 통한 공격에 노출될 수 있습니다.

> ⚠️ **흔한 오해**: "CORS를 설정하면 API가 안전하다"고 생각하는 분들이 많습니다. 하지만 CORS는 **브라우저에서만 적용**됩니다. `curl`이나 Postman 같은 도구, 혹은 서버 간 통신에서는 CORS가 전혀 동작하지 않아요. CORS는 보안의 한 계층일 뿐, 반드시 이번 세션에서 다루는 인증/인가와 함께 사용해야 합니다.

추가로 `TrustedHostMiddleware`를 함께 사용하면 허용된 호스트명 이외의 요청을 서버 레벨에서 차단할 수 있습니다. CORS가 브라우저 측 방어라면, `TrustedHostMiddleware`는 **서버 측 방어**에 해당하죠.

```python
# 인증 시스템과 함께 사용할 때의 CORS 보안 포인트
# (기본 CORS 설정 방법은 17.2 세션 참고)
from starlette.middleware.trustedhost import TrustedHostMiddleware

# 인증 헤더를 허용하면서 출처를 제한
app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://myapp.example.com"],  # 와일드카드 금지
    allow_credentials=True,   # 인증 헤더/쿠키 허용 시 필수
    allow_methods=["POST"],   # LangServe는 POST만 사용
    allow_headers=["Authorization", "Content-Type", "X-API-Key"],
)

# 서버 측 호스트 검증 — CORS와 별개의 방어 계층
app.add_middleware(
    TrustedHostMiddleware,
    allowed_hosts=["myapp.example.com", "*.example.com"],
)
```

### 개념 5: 입력 검증 — 들어오는 짐 검사하기

> 💡 **비유**: 입력 검증은 공항의 **수하물 검사**와 같습니다. 아무리 유효한 탑승권을 가졌더라도, 위험물을 반입할 수는 없죠. API에서도 인증을 통과했더라도, 입력값이 안전한지 별도로 확인해야 합니다.

LangServe는 Pydantic을 통해 입출력 스키마를 자동으로 추론하고 검증합니다. 하지만 기본 스키마만으로는 프롬프트 인젝션이나 과도한 입력 길이 같은 보안 위협을 막기 어려운데요, 커스텀 검증 로직을 추가하여 이를 방어할 수 있습니다.

```python
# input_validation.py — 입력 검증 강화 예제
from typing import Dict

from fastapi import FastAPI, HTTPException, Request
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI
from pydantic import BaseModel, Field, field_validator

from langserve import add_routes

app = FastAPI()

# 입력 길이, 금지어 등을 검증하는 Pydantic 모델
MAX_INPUT_LENGTH = 2000
BLOCKED_PATTERNS = [
    "ignore previous instructions",
    "ignore all instructions",
    "system prompt",
    "you are now",
]


class SafeInput(BaseModel):
    """검증된 안전한 입력 모델"""
    question: str = Field(
        ...,
        min_length=1,
        max_length=MAX_INPUT_LENGTH,
        description="사용자 질문 (최대 2000자)"
    )
    language: str = Field(
        default="ko",
        pattern=r"^(ko|en|ja|zh)$",
        description="응답 언어 코드"
    )

    @field_validator("question")
    @classmethod
    def check_prompt_injection(cls, v: str) -> str:
        """기본적인 프롬프트 인젝션 패턴을 차단합니다."""
        lower_v = v.lower()
        for pattern in BLOCKED_PATTERNS:
            if pattern in lower_v:
                raise ValueError(
                    f"Blocked input pattern detected: {pattern}"
                )
        return v


# with_types로 입력 스키마를 명시적으로 지정
chain = (
    ChatPromptTemplate.from_messages([
        ("system", "You are a helpful assistant. Respond in {language}."),
        ("human", "{question}")
    ])
    | ChatOpenAI(model="gpt-4o", temperature=0.7)
    | StrOutputParser()
).with_types(input_type=SafeInput)  # 커스텀 입력 타입 지정

add_routes(app, chain, path="/safe-chat")

if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000)
```


> ⚠️ **흔한 오해**: Pydantic 입력 검증만으로 프롬프트 인젝션을 완전히 방어할 수 있다고 생각하기 쉽습니다. 하지만 입력 검증은 **1차 방어선**일 뿐이에요. 정교한 인젝션 공격은 패턴 매칭만으로 잡기 어렵기 때문에, 시스템 프롬프트 강화, 출력 필터링, 모니터링 등 **다층 방어(defense-in-depth)**가 필요합니다.

## 실습: 직접 해보기

이제 모든 보안 요소를 결합한 프로덕션급 서버를 만들어 봅시다. API 키 인증, JWT 인증, 입력 검증, 속도 제한을 모두 포함하는 완전한 예제입니다. CORS 설정은 [17.2 FastAPI 통합과 서버 구성](./02-fastapi-통합과-서버-구성.md)에서 배운 내용을 적용하되, 앞서 다룬 인증 관련 보안 포인트를 반영합니다.

```python
# secure_langserve_server.py — 프로덕션급 보안 서버 종합 실습
"""
필요 패키지 설치:
pip install langserve[server] langchain-openai python-jose[cryptography] passlib[bcrypt] slowapi
"""
import os
from datetime import datetime, timedelta, timezone
from typing import Dict, Optional

from fastapi import Depends, FastAPI, HTTPException, Request, status
from fastapi.middleware.cors import CORSMiddleware
from fastapi.security import OAuth2PasswordBearer, OAuth2PasswordRequestForm
from jose import JWTError, jwt
from passlib.context import CryptContext
from pydantic import BaseModel, Field, field_validator
from slowapi import Limiter
from slowapi.util import get_remote_address
from slowapi.errors import RateLimitExceeded
from starlette.responses import JSONResponse
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI
from typing_extensions import Annotated

from langserve import add_routes

# --- 설정 ---
SECRET_KEY = os.getenv("JWT_SECRET_KEY", "change-this-in-production")
ALGORITHM = "HS256"
ACCESS_TOKEN_EXPIRE_MINUTES = 30

# --- 비밀번호 해싱 & OAuth2 ---
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="token")

# --- 속도 제한 설정 ---
limiter = Limiter(key_func=get_remote_address)


# --- 데이터 모델 ---
class Token(BaseModel):
    access_token: str
    token_type: str


class User(BaseModel):
    username: str
    role: str = "user"
    disabled: bool = False


class UserInDB(User):
    hashed_password: str


class ChatInput(BaseModel):
    """검증된 채팅 입력"""
    question: str = Field(
        ..., min_length=1, max_length=2000,
        description="질문 내용"
    )

    @field_validator("question")
    @classmethod
    def validate_question(cls, v: str) -> str:
        blocked = ["ignore previous instructions", "ignore all instructions"]
        lower_v = v.lower()
        for pattern in blocked:
            if pattern in lower_v:
                raise ValueError("Potentially harmful input detected")
        return v.strip()


# --- 사용자 DB (실무에서는 실제 DB 사용) ---
USERS_DB = {
    "alice": UserInDB(
        username="alice", role="admin", disabled=False,
        hashed_password=pwd_context.hash("alice123"),
    ),
    "bob": UserInDB(
        username="bob", role="user", disabled=False,
        hashed_password=pwd_context.hash("bob123"),
    ),
}


# --- 인증 함수 ---
def create_access_token(data: dict) -> str:
    to_encode = data.copy()
    expire = datetime.now(timezone.utc) + timedelta(minutes=ACCESS_TOKEN_EXPIRE_MINUTES)
    to_encode.update({"exp": expire})
    return jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)


async def get_user_from_token(token: str) -> User:
    """JWT 토큰에서 사용자 정보를 추출합니다."""
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        username = payload.get("sub")
        if username is None:
            raise HTTPException(status_code=401, detail="Invalid token")
    except JWTError:
        raise HTTPException(status_code=401, detail="Invalid token")

    user = USERS_DB.get(username)
    if user is None or user.disabled:
        raise HTTPException(status_code=401, detail="User not found or disabled")
    return User(**user.model_dump())


async def per_req_config_modifier(config: Dict, request: Request) -> Dict:
    """요청마다 인증 + 사용자 설정을 주입합니다."""
    auth = request.headers.get("Authorization", "")
    if not auth.startswith("Bearer "):
        raise HTTPException(status_code=401, detail="Bearer token required")

    token = auth.split(" ")[1]
    user = await get_user_from_token(token)

    # 사용자 정보를 configurable에 주입
    config["configurable"] = {
        "user_id": user.username,
        "role": user.role,
    }
    return config


# --- FastAPI 앱 ---
app = FastAPI(
    title="Secure LangServe API",
    description="인증과 보안이 적용된 LangServe API",
    version="1.0.0",
)

# 속도 제한 예외 핸들러
@app.exception_handler(RateLimitExceeded)
async def rate_limit_handler(request: Request, exc: RateLimitExceeded):
    return JSONResponse(
        status_code=429,
        content={"detail": "Too many requests. Please try again later."},
    )

app.state.limiter = limiter

# CORS 보안 설정 (기본 구성은 17.2 참고, 여기서는 인증 헤더 허용에 초점)
app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://myapp.example.com"],  # 인증 사용 시 와일드카드 금지
    allow_credentials=True,
    allow_methods=["POST"],
    allow_headers=["Authorization", "Content-Type"],
)


# --- 엔드포인트 ---
@app.post("/token", response_model=Token)
@limiter.limit("5/minute")  # 로그인 시도 속도 제한
async def login(
    request: Request,
    form_data: Annotated[OAuth2PasswordRequestForm, Depends()],
):
    """로그인 엔드포인트: 사용자 인증 후 JWT 토큰 발급"""
    user = USERS_DB.get(form_data.username)
    if not user or not pwd_context.verify(form_data.password, user.hashed_password):
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="Incorrect username or password",
        )
    token = create_access_token({"sub": user.username, "role": user.role})
    return Token(access_token=token, token_type="bearer")


@app.get("/me")
async def get_me(token: Annotated[str, Depends(oauth2_scheme)]):
    """현재 인증된 사용자 정보를 반환합니다."""
    user = await get_user_from_token(token)
    return {"username": user.username, "role": user.role}


# 체인 정의
chain = (
    ChatPromptTemplate.from_messages([
        ("system", "You are a helpful assistant. Answer concisely in Korean."),
        ("human", "{question}"),
    ])
    | ChatOpenAI(model="gpt-4o", temperature=0.7)
    | StrOutputParser()
).with_types(input_type=ChatInput)

# 보안이 적용된 라우트 등록
add_routes(
    app,
    chain,
    path="/chat",
    per_req_config_modifier=per_req_config_modifier,  # 사용자별 설정
    enabled_endpoints=["invoke", "stream"],            # 필요한 엔드포인트만 노출
)

if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000)
```

## 더 깊이 알아보기

### JWT의 탄생 — 쿠키의 한계를 넘어서

JWT(JSON Web Token)는 2010년대 초, 웹 인증의 패러다임이 변화하던 시기에 탄생했습니다. 전통적인 웹 인증은 **세션 쿠키** 방식이었는데요 — 서버가 세션 ID를 메모리에 저장하고, 클라이언트에 쿠키로 전달하는 방식이었죠.

하지만 모바일 앱과 마이크로서비스가 등장하면서 문제가 생겼습니다. 쿠키는 웹 브라우저 전용이고, 세션 데이터를 서버 메모리에 저장하면 서버를 여러 대로 확장(scale-out)할 때 세션 공유가 골칫거리였거든요.

**Mike Jones, John Bradley, Nat Sakimura** — 이 세 명의 엔지니어가 OAuth 워킹 그룹에서 해결책을 고안했습니다. "인증 정보를 서버가 아닌 **토큰 자체에 담으면** 어떨까?" 이 아이디어가 JWT의 핵심입니다. 4년 반의 표준화 과정을 거쳐 2015년 5월, **RFC 7519**로 공식 발표되었죠.

JWT는 서버가 상태를 보관하지 않아도 되는(stateless) 특성 덕분에, 오늘날 REST API 인증의 사실상 표준이 되었습니다. LangServe처럼 LLM API를 서빙하는 환경에서 특히 유용한데, 여러 워커가 동시에 요청을 처리해도 세션 저장소를 공유할 필요가 없기 때문입니다.

### LangServe의 보안 철학 — "우리는 프레임워크를 제공하지, 보안 솔루션을 제공하지 않는다"

LangServe가 자체적인 인증 시스템을 내장하지 않은 것은 의도적인 설계 결정입니다. FastAPI라는 검증된 프레임워크의 보안 생태계를 그대로 활용하도록 했죠. 이는 "바퀴를 다시 발명하지 말라"는 소프트웨어 엔지니어링 원칙과도 맞닿아 있습니다. FastAPI의 `Depends`, 미들웨어, `OAuth2PasswordBearer` 같은 보안 기능은 수년간 검증된 것이니까요.

## 흔한 오해와 팁

> 💡 **알고 계셨나요?**: JWT의 `exp`(만료 시간) 클레임을 짧게 설정하면 보안이 강화되지만, 사용자 경험이 나빠집니다. 반대로 너무 길면 토큰 탈취 시 위험합니다. 실무에서 흔히 사용하는 패턴은 **Access Token은 15~30분**, **Refresh Token은 7일**로 설정하고, Access Token이 만료되면 Refresh Token으로 자동 갱신하는 것입니다.

> 🔥 **실무 팁**: `per_req_config_modifier`에서 추출한 사용자 정보(토큰, 사용자 ID 등)는 `config["configurable"]`에만 저장하세요. 이 정보가 절대 프롬프트 입력에 섞이지 않도록 해야 합니다. 만약 사용자 토큰이 프롬프트에 들어가면, LLM이 응답에 토큰을 포함시킬 위험이 있거든요. 이것이 LangServe 공식 예제가 `configurable`을 사용하는 핵심 이유입니다.

> 🔥 **실무 팁**: `enabled_endpoints` 매개변수로 꼭 필요한 엔드포인트만 노출하세요. 예를 들어, 스트리밍만 지원할 API라면 `enabled_endpoints=["stream"]`으로 제한하면 불필요한 공격 표면을 줄일 수 있습니다.

## 핵심 정리

| 개념 | 설명 |
|------|------|
| **API 키 인증** | `dependencies=[Depends(verify_api_key)]`로 헤더 기반 간단 인증 |
| **JWT 토큰 인증** | `python-jose`로 토큰 발급·검증, 사용자 식별 + 권한 관리 가능 |
| **per_req_config_modifier** | 요청마다 인증 정보를 `config["configurable"]`에 주입하여 사용자별 로직 구현 |
| **CORS + 인증** | 인증 헤더 허용 시 `allow_credentials=True` + 명시적 출처 필수 ([17.2](./02-fastapi-통합과-서버-구성.md) 참고) |
| **입력 검증** | Pydantic `field_validator`와 `with_types`로 길이·패턴·인젝션 방어 |
| **속도 제한** | `slowapi`로 IP 기반 요청 횟수 제한, 특히 로그인 엔드포인트에 필수 |
| **enabled_endpoints** | 필요한 엔드포인트만 노출하여 공격 표면 최소화 |

## 다음 섹션 미리보기

인증과 보안으로 API를 보호하는 방법을 익혔으니, 다음 세션에서는 **클라이언트 SDK와 스트리밍**을 다룹니다. `RemoteRunnable`을 활용한 고급 호출 패턴, 스트리밍 응답 처리, 에러 핸들링과 재시도 전략, 비동기 배치 호출까지 — 서버를 만들었으니 이제 클라이언트 측에서 이 API를 효과적으로 사용하는 방법을 배워봅니다.
