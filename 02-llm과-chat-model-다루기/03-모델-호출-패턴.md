# 모델 호출 패턴: invoke, stream, batch와 비동기 처리

> LangChain의 Runnable 인터페이스가 제공하는 다양한 호출 방식을 마스터하고, 실시간 스트리밍부터 대량 배치 처리, 비동기 동시성 제어까지 프로덕션에 필요한 모든 호출 패턴을 익힙니다.

## 개요

이 섹션에서는 LangChain의 모든 컴포넌트가 공유하는 **Runnable 인터페이스**의 핵심 호출 메서드들을 깊이 있게 다룹니다. 단일 호출(`invoke`)부터 실시간 스트리밍(`stream`), 대량 배치 처리(`batch`), 그리고 비동기 버전(`ainvoke`, `astream`, `abatch`)까지 — 상황에 맞는 최적의 호출 패턴을 선택할 수 있게 됩니다.

**선수 지식**: [Session 2.1: ChatOpenAI 심화](ch02/session_2_1.md)에서 배운 ChatOpenAI 초기화와 파라미터 설정, [Session 2.2: 다중 프로바이더 연동](ch02/session_2_2.md)에서 배운 다양한 Chat Model 프로바이더 사용법

**학습 목표**:
- `invoke`, `stream`, `batch`의 차이점과 각각의 사용 시나리오를 이해한다
- `ainvoke`, `astream`, `abatch`로 비동기 호출을 구현한다
- `max_concurrency`와 `InMemoryRateLimiter`로 동시성과 요청 속도를 제어한다
- `batch_as_completed`로 결과가 준비되는 대로 처리하는 패턴을 구현한다

## 왜 알아야 할까?

여러분이 레스토랑을 운영한다고 상상해 보세요. 손님이 한 명 오면 주문을 받고 요리를 내보내면 됩니다(`invoke`). 하지만 점심시간에 30명이 한꺼번에 들어온다면요? 한 명씩 순서대로 처리하면 마지막 손님은 영원히 기다려야 합니다. 주문을 모아서 한꺼번에 주방에 넘기거나(`batch`), 요리가 나오는 대로 바로 서빙하거나(`stream`), 여러 셰프가 동시에 요리하게 해야 하죠(`async`).

LLM 애플리케이션도 마찬가지입니다. 챗봇은 글자가 하나씩 나타나야 사용자가 답답하지 않고, 문서 100개를 요약할 때는 배치 처리가 효율적이며, 웹 서버에서 여러 사용자 요청을 동시에 처리하려면 비동기가 필수입니다. 앞서 [Session 2.2](ch02/session_2_2.md)에서 프로바이더를 자유롭게 전환하는 법을 배웠는데, 어떤 프로바이더를 쓰든 이 호출 패턴은 **동일하게** 적용됩니다. 이것이 바로 Runnable 인터페이스의 힘이거든요.

## 핵심 개념

### 개념 1: Runnable 인터페이스 — 모든 것의 기반

> 💡 **비유**: Runnable 인터페이스는 USB-C 포트와 같습니다. 충전기든, 외장 하드든, 모니터든 — USB-C라는 하나의 규격만 알면 모든 기기를 연결할 수 있죠. LangChain에서도 프롬프트, 모델, 파서 등 모든 컴포넌트가 Runnable이라는 하나의 인터페이스를 공유합니다.

LangChain의 모든 핵심 컴포넌트 — ChatModel, PromptTemplate, OutputParser, Retriever 등 — 는 `Runnable` 프로토콜을 구현합니다. 이 프로토콜이 보장하는 표준 메서드는 다음과 같습니다:

| 메서드 | 동기 | 비동기 | 설명 |
|--------|------|--------|------|
| 단일 호출 | `invoke()` | `ainvoke()` | 입력 하나 → 출력 하나 |
| 스트리밍 | `stream()` | `astream()` | 입력 하나 → 출력을 청크 단위로 |
| 배치 | `batch()` | `abatch()` | 입력 여러 개 → 출력 여러 개 |
| 배치(완료순) | `batch_as_completed()` | `abatch_as_completed()` | 입력 여러 개 → 완료되는 순서대로 |

이 모든 메서드는 선택적으로 `config` 딕셔너리를 받아 태그, 메타데이터, `max_concurrency` 등을 설정할 수 있습니다.

```python
from langchain_openai import ChatOpenAI

# 어떤 Runnable이든 동일한 메서드를 제공합니다
model = ChatOpenAI(model="gpt-4o", temperature=0.7)

# 단일 호출
result = model.invoke("안녕하세요!")

# 스트리밍
for chunk in model.stream("안녕하세요!"):
    print(chunk.content, end="")

# 배치
results = model.batch(["안녕!", "반가워!", "잘 가!"])
```

### 개념 2: invoke — 기본이자 가장 중요한 호출

> 💡 **비유**: `invoke`는 편의점에서 물건 하나 사는 것과 같습니다. 물건을 고르고(`input`), 계산하고(`처리`), 영수증을 받는(`output`) 가장 단순한 거래죠.

`invoke()`는 하나의 입력을 받아 하나의 출력을 반환합니다. 모든 호출 패턴의 기초이며, 가장 자주 사용하는 메서드입니다.

```python
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, SystemMessage

model = ChatOpenAI(model="gpt-4o", temperature=0.7)

# 문자열로 호출 — 내부적으로 HumanMessage로 변환됩니다
response = model.invoke("Python의 장점을 3가지 알려주세요")
print(response.content)

# 메시지 리스트로 호출 — 시스템 메시지 포함 가능
messages = [
    SystemMessage(content="당신은 Python 전문가입니다. 간결하게 답변하세요."),
    HumanMessage(content="Python의 장점을 3가지 알려주세요"),
]
response = model.invoke(messages)
print(response.content)

# config 옵션 전달 — 태그, 메타데이터 등
response = model.invoke(
    "Python의 장점을 알려주세요",
    config={
        "tags": ["python-qa"],           # 추적용 태그
        "metadata": {"user_id": "u123"}, # 메타데이터
    },
)
```

`invoke()`의 반환값은 `AIMessage` 객체로, `.content`에 텍스트 응답이, `.response_metadata`에 모델 메타데이터가, `.usage_metadata`에 토큰 사용량 정보가 담겨 있습니다(Session 2.1에서 배운 내용이죠).

### 개념 3: stream — 실시간 스트리밍

> 💡 **비유**: `stream`은 유튜브 영상 시청과 같습니다. 영상 전체가 다운로드될 때까지 기다리지 않고, 받는 족족 재생하죠. LLM 응답도 마찬가지로 — 전체 답변이 완성될 때까지 기다리지 않고, 생성되는 토큰을 즉시 화면에 표시합니다.

`stream()`은 LLM이 토큰을 생성할 때마다 `AIMessageChunk` 객체를 하나씩 yield합니다. 사용자 경험 측면에서 ChatGPT처럼 글자가 하나씩 나타나는 효과를 구현할 수 있어요.

```python
from langchain_openai import ChatOpenAI

model = ChatOpenAI(model="gpt-4o", temperature=0.7)

# 기본 스트리밍 — 토큰이 생성될 때마다 출력
print("응답: ", end="")
for chunk in model.stream("머신러닝을 비유로 쉽게 설명해주세요"):
    print(chunk.content, end="", flush=True)
    # flush=True: 버퍼링 없이 즉시 출력
print()  # 줄바꿈

# 청크에서 메타데이터 수집하기
all_chunks = []
for chunk in model.stream("짧게 인사해주세요"):
    all_chunks.append(chunk)
    print(chunk.content, end="", flush=True)

# 청크를 합치면 완전한 AIMessage가 됩니다
full_message = all_chunks[0]
for chunk in all_chunks[1:]:
    full_message = full_message + chunk  # + 연산자로 청크 합치기

print(f"\n\n전체 응답: {full_message.content}")
print(f"총 청크 수: {len(all_chunks)}")
```

> ⚠️ **흔한 오해**: "스트리밍은 전체 응답보다 느리다"고 생각하는 분이 많은데요, 사실 **총 처리 시간은 거의 동일**합니다. 스트리밍의 장점은 **첫 번째 토큰까지의 시간(TTFT, Time To First Token)**이 매우 짧다는 것이에요. 사용자는 즉시 응답이 시작되는 것을 보기 때문에 체감 속도가 훨씬 빠릅니다.

### 개념 4: batch — 대량 처리의 효율성

> 💡 **비유**: `batch`는 세탁기에 빨래를 모아서 돌리는 것과 같습니다. 양말 하나씩 세탁하는 것보다 모아서 한 번에 돌리는 게 훨씬 효율적이죠. 100개의 질문을 하나씩 `invoke`로 보내면 네트워크 오버헤드만 100번 발생하지만, `batch`로 보내면 스레드 풀을 사용해 병렬로 처리합니다.

`batch()`는 입력 리스트를 받아 내부적으로 `ThreadPoolExecutor`를 사용해 여러 `invoke()` 호출을 병렬로 실행합니다.

```python
import time
from langchain_openai import ChatOpenAI

model = ChatOpenAI(model="gpt-4o", temperature=0.7)

questions = [
    "Python이란 무엇인가요?",
    "JavaScript란 무엇인가요?",
    "Rust란 무엇인가요?",
    "Go란 무엇인가요?",
    "TypeScript란 무엇인가요?",
]

# 순차 처리 — invoke를 반복 호출
start = time.time()
sequential_results = [model.invoke(q) for q in questions]
sequential_time = time.time() - start
print(f"순차 처리: {sequential_time:.2f}초")

# 배치 처리 — batch로 병렬 호출
start = time.time()
batch_results = model.batch(questions)
batch_time = time.time() - start
print(f"배치 처리: {batch_time:.2f}초")

print(f"속도 향상: {sequential_time / batch_time:.1f}배")

# 결과 확인
for q, r in zip(questions, batch_results):
    print(f"\nQ: {q}")
    print(f"A: {r.content[:80]}...")
```

#### max_concurrency로 동시성 제어

API 프로바이더마다 분당 요청 수(RPM) 제한이 있습니다. `batch()`에 `max_concurrency` 옵션을 전달하면 동시 실행 수를 제한할 수 있습니다.

```python
# 동시에 최대 3개까지만 병렬 실행
results = model.batch(
    questions,
    config={"max_concurrency": 3},
)

# 이렇게 하면:
# - 처음 3개 요청이 동시에 실행
# - 하나가 완료되면 다음 요청이 시작
# - API rate limit 초과를 방지
```

#### batch_as_completed — 완료 순서대로 처리

일반 `batch()`는 모든 결과가 준비될 때까지 기다리지만, `batch_as_completed()`는 먼저 완료된 결과부터 반환합니다. 긴 응답과 짧은 응답이 섞여 있을 때 유용하죠.

```python
from langchain_openai import ChatOpenAI

model = ChatOpenAI(model="gpt-4o", temperature=0.7)

questions = [
    "양자역학의 기본 원리를 설명해주세요",        # 긴 응답 예상
    "1+1은?",                                    # 짧은 응답
    "대한민국의 수도는?",                         # 짧은 응답
    "인공지능의 역사를 시대별로 정리해주세요",     # 긴 응답 예상
]

# 완료되는 순서대로 결과를 받습니다
for idx, result in model.batch_as_completed(questions):
    # idx: 원래 입력 리스트에서의 인덱스
    print(f"[완료 #{idx}] Q: {questions[idx]}")
    print(f"         A: {result.content[:60]}...\n")
```

### 개념 5: 비동기 호출 — ainvoke, astream, abatch

> 💡 **비유**: 동기(sync) 호출은 전화 통화와 같습니다 — 상대방이 말할 때까지 기다려야 하죠. 비동기(async) 호출은 메신저와 같아요 — 메시지를 보내고 답이 올 때까지 다른 일을 할 수 있습니다. 웹 서버가 100명의 사용자를 동시에 처리하려면 이 "메신저 방식"이 필수입니다.

Python의 `asyncio`를 활용한 비동기 메서드들은 특히 웹 애플리케이션에서 핵심적인 역할을 합니다. FastAPI, LangServe 같은 비동기 프레임워크와 함께 사용할 때 진가를 발휘하거든요.

```python
import asyncio
import time
from langchain_openai import ChatOpenAI

model = ChatOpenAI(model="gpt-4o", temperature=0.7)

# --- ainvoke: 비동기 단일 호출 ---
async def async_invoke_example():
    response = await model.ainvoke("Python의 장점은?")
    print(f"ainvoke 결과: {response.content[:80]}...")

# --- astream: 비동기 스트리밍 ---
async def async_stream_example():
    print("astream 결과: ", end="")
    async for chunk in model.astream("짧게 인사해주세요"):
        print(chunk.content, end="", flush=True)
    print()

# --- abatch: 비동기 배치 ---
async def async_batch_example():
    questions = ["Python이란?", "Java란?", "Rust란?"]
    results = await model.abatch(questions)
    for q, r in zip(questions, results):
        print(f"Q: {q} → A: {r.content[:50]}...")

# 모든 비동기 함수 실행
async def main():
    await async_invoke_example()
    print("---")
    await async_stream_example()
    print("---")
    await async_batch_example()

asyncio.run(main())
```

#### 비동기의 진짜 위력: 동시 실행

비동기의 핵심은 **여러 작업을 동시에** 실행할 수 있다는 것입니다. `asyncio.gather()`를 사용하면 여러 `ainvoke` 호출을 동시에 실행하고, 모든 결과가 모일 때까지 기다릴 수 있습니다.

```python
import asyncio
import time
from langchain_openai import ChatOpenAI

model = ChatOpenAI(model="gpt-4o", temperature=0.7)

questions = [
    "Python의 특징을 3줄로",
    "JavaScript의 특징을 3줄로",
    "Rust의 특징을 3줄로",
    "Go의 특징을 3줄로",
]

async def sequential_async():
    """비동기이지만 순차 실행 — 이래선 비동기 의미가 없죠!"""
    start = time.time()
    results = []
    for q in questions:
        result = await model.ainvoke(q)
        results.append(result)
    print(f"순차 비동기: {time.time() - start:.2f}초")
    return results

async def concurrent_async():
    """asyncio.gather로 동시 실행 — 이것이 비동기의 진짜 힘!"""
    start = time.time()
    results = await asyncio.gather(
        *[model.ainvoke(q) for q in questions]
    )
    print(f"동시 비동기: {time.time() - start:.2f}초")
    return results

async def main():
    await sequential_async()
    await concurrent_async()

asyncio.run(main())
# 출력 예시:
# 순차 비동기: 8.23초
# 동시 비동기: 2.51초
```

### 개념 6: InMemoryRateLimiter — 요청 속도 제어

> 💡 **비유**: Rate Limiter는 놀이공원의 입장 게이트와 같습니다. 한꺼번에 너무 많은 사람이 들어가면 사고가 나니까, 일정한 속도로만 입장을 허용하죠. API도 마찬가지로 — 초당 허용되는 요청 수를 넘으면 에러가 발생합니다.

LangChain은 `InMemoryRateLimiter`를 내장으로 제공합니다. 토큰 버킷(Token Bucket) 알고리즘 기반으로, 모델 초기화 시 `rate_limiter` 파라미터에 전달하면 됩니다.

```python
import time
from langchain_core.rate_limiters import InMemoryRateLimiter
from langchain_openai import ChatOpenAI

# 초당 1회 요청으로 제한하는 Rate Limiter 생성
rate_limiter = InMemoryRateLimiter(
    requests_per_second=1.0,       # 초당 허용 요청 수
    check_every_n_seconds=0.1,     # 토큰 확인 주기 (100ms마다)
    max_bucket_size=5,             # 최대 버스트 크기
)

# 모델에 rate_limiter 적용
model = ChatOpenAI(
    model="gpt-4o-mini",
    temperature=0,
    rate_limiter=rate_limiter,     # 여기서 적용!
)

# rate limiter가 요청 속도를 자동으로 조절합니다
questions = [f"숫자 {i}를 영어로 쓰면?" for i in range(5)]

start = time.time()
for i, q in enumerate(questions):
    result = model.invoke(q)
    elapsed = time.time() - start
    print(f"[{elapsed:.1f}초] Q: {q} → A: {result.content}")
# 출력 예시:
# [0.8초] Q: 숫자 0를 영어로 쓰면? → A: zero
# [1.9초] Q: 숫자 1를 영어로 쓰면? → A: one
# [3.0초] Q: 숫자 2를 영어로 쓰면? → A: two
# ...초당 1회씩 실행됩니다
```

`InMemoryRateLimiter`의 주요 파라미터를 정리하면:

| 파라미터 | 설명 | 기본값 |
|---------|------|--------|
| `requests_per_second` | 초당 허용 요청 수 | 1 |
| `check_every_n_seconds` | 토큰 가용 여부 확인 주기 | 0.1 |
| `max_bucket_size` | 버스트 허용 최대 크기 | 10 |

> 🔥 **실무 팁**: `InMemoryRateLimiter`는 **스레드 안전(thread-safe)** 하기 때문에 여러 스레드가 같은 rate limiter를 공유할 수 있습니다. 단, **프로세스 간 공유는 불가능**하므로 멀티프로세스 환경에서는 Redis 기반 rate limiter 등 별도 솔루션이 필요합니다.

#### max_concurrency와 InMemoryRateLimiter 조합

프로덕션에서는 `max_concurrency`와 `InMemoryRateLimiter`를 함께 사용하여 동시성과 요청 속도를 이중으로 제어하는 것이 좋습니다.

```python
from langchain_core.rate_limiters import InMemoryRateLimiter
from langchain_openai import ChatOpenAI

rate_limiter = InMemoryRateLimiter(
    requests_per_second=2.0,   # 초당 2회
    max_bucket_size=5,
)

model = ChatOpenAI(
    model="gpt-4o-mini",
    rate_limiter=rate_limiter,
)

questions = [f"{i}번째 소수는?" for i in range(1, 11)]

# max_concurrency로 동시 실행 수 제한 + rate_limiter로 초당 속도 제한
results = model.batch(
    questions,
    config={"max_concurrency": 3},  # 동시에 3개까지만
)

for q, r in zip(questions, results):
    print(f"Q: {q} → A: {r.content[:30]}")
```

## 실습: 직접 해보기

다양한 호출 패턴을 비교하고, 실전에서 활용할 수 있는 뉴스 요약 파이프라인을 구축해 봅시다.

```python
"""
실습: 뉴스 기사 배치 요약 + 실시간 스트리밍 리포트 생성
- invoke, stream, batch, ainvoke를 모두 활용합니다
- 실행 전: pip install langchain-openai python-dotenv
- .env 파일에 OPENAI_API_KEY 설정 필요
"""

import asyncio
import time
from dotenv import load_dotenv
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, SystemMessage
from langchain_core.rate_limiters import InMemoryRateLimiter

load_dotenv()

# --- 설정 ---
MODEL_NAME = "gpt-4o-mini"
TEMPERATURE = 0.3

# Rate limiter 설정 (초당 2회 제한)
rate_limiter = InMemoryRateLimiter(
    requests_per_second=2.0,
    check_every_n_seconds=0.1,
    max_bucket_size=5,
)

model = ChatOpenAI(
    model=MODEL_NAME,
    temperature=TEMPERATURE,
    rate_limiter=rate_limiter,
)


# --- 1단계: 뉴스 기사들 (시뮬레이션 데이터) ---
news_articles = [
    "오늘 정부는 AI 산업 육성을 위해 5년간 1조원을 투자하는 계획을 발표했다. "
    "반도체, 클라우드, AI 모델 개발에 집중 투자하며, 연구 인력 10만 명 양성을 목표로 한다.",

    "글로벌 전기차 시장이 2025년 사상 최대 판매량을 기록했다. "
    "중국 업체들의 가격 경쟁력이 시장 성장을 주도하고 있으며, 유럽과 미국도 보조금을 확대 중이다.",

    "새로운 양자 컴퓨터가 기존 슈퍼컴퓨터 대비 100만 배 빠른 연산 속도를 달성했다. "
    "연구팀은 오류 정정 기술의 획기적 발전이 핵심이었다고 밝혔다.",

    "원격근무 확산으로 지방 소도시의 인구가 증가하고 있다. "
    "IT 기업 종사자들을 중심으로 수도권 탈출이 이어지며 지역 경제가 활성화되고 있다.",

    "최근 연구에 따르면 하루 30분의 걷기 운동만으로도 심혈관 질환 위험을 40% 줄일 수 있다. "
    "전문가들은 강도보다 꾸준함이 중요하다고 강조했다.",
]


# --- 2단계: batch로 기사 요약 (병렬 처리) ---
def batch_summarize(articles: list[str]) -> list[str]:
    """batch()를 사용해 여러 기사를 병렬로 요약합니다."""
    system_msg = SystemMessage(
        content="당신은 뉴스 요약 전문가입니다. 기사를 한 문장으로 요약하세요."
    )
    inputs = [
        [system_msg, HumanMessage(content=article)]
        for article in articles
    ]

    start = time.time()
    results = model.batch(
        inputs,
        config={"max_concurrency": 3},  # 동시에 3개까지 병렬 실행
    )
    elapsed = time.time() - start

    summaries = [r.content for r in results]
    print(f"\n[배치 요약] {len(articles)}개 기사를 {elapsed:.2f}초에 처리")
    return summaries


# --- 3단계: stream으로 종합 리포트 생성 (실시간 출력) ---
def stream_report(summaries: list[str]):
    """stream()으로 종합 리포트를 실시간으로 생성합니다."""
    summaries_text = "\n".join(
        f"{i+1}. {s}" for i, s in enumerate(summaries)
    )

    messages = [
        SystemMessage(
            content="당신은 뉴스 브리핑 앵커입니다. "
            "요약된 뉴스들을 자연스럽게 연결하여 "
            "오늘의 뉴스 브리핑을 작성하세요."
        ),
        HumanMessage(
            content=f"다음 뉴스 요약들로 브리핑을 작성해주세요:\n\n{summaries_text}"
        ),
    ]

    print("\n📺 [오늘의 뉴스 브리핑 — 실시간 스트리밍]")
    print("=" * 50)
    for chunk in model.stream(messages):
        print(chunk.content, end="", flush=True)
    print("\n" + "=" * 50)


# --- 4단계: 비동기로 추가 분석 동시 실행 ---
async def async_analyze(summaries: list[str]):
    """ainvoke를 asyncio.gather로 동시 실행하여 다각도 분석을 수행합니다."""
    analysis_prompts = [
        f"다음 뉴스 요약에서 긍정적 시사점을 하나 뽑아주세요: {summaries[0]}",
        f"다음 뉴스 요약에서 기술 트렌드를 분석해주세요: {summaries[2]}",
        f"다음 뉴스 요약의 사회적 영향을 한 줄로 분석해주세요: {summaries[3]}",
    ]

    start = time.time()
    results = await asyncio.gather(
        *[model.ainvoke(prompt) for prompt in analysis_prompts]
    )
    elapsed = time.time() - start

    print(f"\n[비동기 분석] 3개 분석을 {elapsed:.2f}초에 동시 완료")
    for i, result in enumerate(results):
        print(f"  분석 {i+1}: {result.content[:80]}...")


# --- 실행 ---
def main():
    print("🚀 뉴스 요약 파이프라인 시작\n")

    # 1) batch: 기사 요약
    summaries = batch_summarize(news_articles)
    for i, s in enumerate(summaries):
        print(f"  [{i+1}] {s}")

    # 2) stream: 종합 브리핑 생성
    stream_report(summaries)

    # 3) async: 다각도 분석 동시 실행
    asyncio.run(async_analyze(summaries))

    print("\n✅ 파이프라인 완료!")


if __name__ == "__main__":
    main()
```

## 더 깊이 알아보기

### asyncio의 탄생 — 파이썬 비동기의 대장정

Python의 `asyncio`는 파이썬 창시자 **Guido van Rossum**이 직접 설계한 비동기 프레임워크입니다. 2012년 12월, Guido는 [PEP 3156](https://peps.python.org/pep-3156/)을 제출하면서 "Asynchronous IO Support Rebooted"라는 제목을 붙였는데요, "Rebooted"라는 단어에서 알 수 있듯이 이전에도 비동기 시도가 있었지만 파편화되어 있었거든요.

당시 Python 생태계에는 Twisted, Tornado, gevent 등 여러 비동기 라이브러리가 난립하고 있었습니다. 각각 장단점이 있었지만 서로 호환되지 않아 개발자들이 고통받고 있었죠. Guido는 Twisted와 Tornado 개발자들과 협력하여 **통합된 이벤트 루프**를 설계했고, 2014년 Python 3.4에 `asyncio`가 "provisional" 상태로 포함되었습니다. 그리고 Python 3.5에서 `async/await` 키워드가 도입되면서 비동기 프로그래밍이 비로소 시민권을 얻게 됩니다.

재미있는 것은 `asyncio`의 내부 코드명이 **"Tulip"**(튤립)이었다는 점인데요, Guido가 네덜란드 출신이라 네덜란드의 국화인 튤립에서 이름을 따온 것이라고 합니다.

LangChain이 모든 Runnable에 `ainvoke`, `astream`, `abatch` 같은 비동기 메서드를 제공하는 것도 이 `asyncio` 생태계 위에서 가능한 것이죠. 비동기의 기본 구현은 동기 메서드를 `asyncio`의 스레드 풀에서 실행하는 방식이지만, ChatOpenAI 같은 주요 모델들은 httpx 기반의 **네이티브 비동기** 구현을 제공하여 성능이 훨씬 우수합니다.

### 토큰 버킷 알고리즘 — Rate Limiter의 원리

`InMemoryRateLimiter`가 사용하는 **토큰 버킷(Token Bucket)** 알고리즘은 1980년대 네트워크 트래픽 제어를 위해 고안되었습니다. 원리는 놀랍도록 단순합니다:

1. 일정 속도로 버킷에 토큰이 채워집니다 (`requests_per_second`)
2. 요청할 때마다 버킷에서 토큰 하나를 꺼냅니다
3. 버킷이 비어 있으면 토큰이 채워질 때까지 대기합니다
4. 버킷에는 최대 크기가 있어서 토큰이 무한히 쌓이지 않습니다 (`max_bucket_size`)

이 알고리즘의 우아한 점은 **평균 속도를 유지하면서도 짧은 버스트를 허용**한다는 것입니다. `max_bucket_size=5`이면 토큰이 5개까지 쌓일 수 있어서, 잠깐 쉬었다가 한꺼번에 5개를 빠르게 보내는 것이 가능하거든요.

## 흔한 오해와 팁

> ⚠️ **흔한 오해**: "비동기 코드는 항상 동기 코드보다 빠르다." — 그렇지 않습니다! `async`가 빛을 발하는 건 **I/O 대기 시간이 긴 작업을 여러 개 동시에 할 때**입니다. 단일 API 호출 하나만 한다면 동기 `invoke`와 비동기 `ainvoke`의 속도 차이는 거의 없어요. 비동기의 진짜 가치는 `asyncio.gather`로 여러 호출을 **동시에** 날릴 때 드러납니다.

> 💡 **알고 계셨나요?**: LangChain의 LCEL(파이프 연산자 `|`)로 체인을 구성하면, `invoke`, `stream`, `batch`, `ainvoke` 등 **모든 호출 패턴이 자동으로** 지원됩니다. 체인의 각 컴포넌트가 Runnable이기 때문에 별도 코드 변경 없이 `chain.stream()`을 호출하면 마지막 컴포넌트의 출력이 스트리밍됩니다. 이는 [Session 5장: LCEL 마스터](ch05/session_5_1.md)에서 더 자세히 다룰 예정입니다.

> 🔥 **실무 팁**: `batch()` 사용 시 `return_exceptions=True` 옵션을 활용하세요. 이 옵션을 켜면 일부 요청이 실패해도 나머지 결과는 정상적으로 반환됩니다. 기본값(`False`)에서는 하나라도 실패하면 예외가 발생하여 전체 배치가 중단됩니다.
>
> ```python
> # 하나의 실패가 전체를 중단시키지 않음
> results = model.batch(
>     questions,
>     return_exceptions=True,  # 실패한 항목은 Exception 객체로 반환
> )
> for i, r in enumerate(results):
>     if isinstance(r, Exception):
>         print(f"[{i}] 실패: {r}")
>     else:
>         print(f"[{i}] 성공: {r.content[:50]}...")
> ```

> 🔥 **실무 팁**: Jupyter Notebook에서 `asyncio.run()`을 사용하면 "이미 이벤트 루프가 실행 중" 에러가 발생합니다. Notebook에서는 대신 `await` 키워드를 직접 사용하거나, `nest_asyncio` 패키지를 설치하세요.
>
> ```python
> # Jupyter Notebook에서
> import nest_asyncio
> nest_asyncio.apply()
>
> # 이제 asyncio.run()도 정상 작동
> asyncio.run(main())
>
> # 또는 await를 직접 사용 (Notebook 셀에서)
> result = await model.ainvoke("안녕하세요!")
> ```

## 핵심 정리

| 개념 | 설명 |
|------|------|
| `invoke()` | 단일 입력 → 단일 출력. 가장 기본적인 호출 방식 |
| `stream()` | 토큰 단위 실시간 출력. TTFT가 빨라 UX 향상에 핵심 |
| `batch()` | 입력 리스트를 ThreadPoolExecutor로 병렬 처리 |
| `batch_as_completed()` | 완료되는 순서대로 (인덱스, 결과) 튜플 반환 |
| `ainvoke/astream/abatch` | asyncio 기반 비동기 버전. 웹 서버 등 동시성 필요 시 사용 |
| `asyncio.gather` | 여러 비동기 호출을 동시에 실행하는 핵심 패턴 |
| `max_concurrency` | `config`에서 동시 실행 수를 제한. batch와 함께 사용 |
| `InMemoryRateLimiter` | 토큰 버킷 알고리즘 기반 요청 속도 제한기. 스레드 안전 |
| `return_exceptions` | batch에서 실패한 항목을 Exception으로 반환하여 전체 중단 방지 |
