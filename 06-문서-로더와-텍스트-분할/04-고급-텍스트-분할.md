# 고급 텍스트 분할

> 문서의 구조와 의미를 이해하는 지능적 텍스트 분할 전략을 마스터합니다.

## 개요

이 섹션에서는 앞서 [6.3: 텍스트 분할 전략](ch06/session_03.md)에서 배운 `RecursiveCharacterTextSplitter`를 넘어, **문서의 구조와 의미를 인식하는 고급 분할기**들을 학습합니다. 마크다운 헤더, HTML 태그, 프로그래밍 언어 문법, 그리고 임베딩 기반 의미 분석까지 — 텍스트의 본질적인 구조를 존중하는 분할 전략을 익힙니다. [6.3](ch06/session_03.md)에서 소개한 Greg Kamradt의 "5 Levels of Text Splitting" 프레임워크 중, 이 세션에서는 **Level 3-4에 해당하는 전략**을 집중적으로 다룹니다.

**선수 지식**: [6.3: 텍스트 분할 전략](ch06/session_03.md)에서 배운 `chunk_size`, `chunk_overlap`, `RecursiveCharacterTextSplitter`의 동작 원리
**학습 목표**:
- `MarkdownHeaderTextSplitter`로 마크다운 문서의 계층 구조를 보존하며 분할할 수 있다
- `HTMLHeaderTextSplitter`로 HTML 문서를 헤더 기준으로 구조적 분할할 수 있다
- `Language` enum과 `from_language()`로 소스 코드를 언어별 문법에 맞게 분할할 수 있다
- `SemanticChunker`의 의미 기반 분할 원리를 직관적으로 이해하고, 기본적인 사용법을 익힐 수 있다

## 왜 알아야 할까?

앞서 배운 `RecursiveCharacterTextSplitter`는 범용적이고 강력하지만, 한 가지 한계가 있습니다 — **문서의 구조를 모른다**는 거죠. 마크다운 문서의 `## 제목`이 의미하는 바를 이해하지 못하고, Python 코드에서 `def` 키워드가 함수의 시작이라는 것도 알지 못합니다.

실무에서 RAG 시스템을 구축하다 보면, 이런 상황을 자주 마주치게 됩니다:

- **기술 문서**: 마크다운으로 작성된 API 문서에서 각 섹션이 독립적인 개념을 설명하는데, 글자 수 기준으로 자르면 하나의 개념이 두 청크에 걸쳐버립니다
- **웹 크롤링 데이터**: HTML 페이지를 가져왔는데, `<h2>` 태그로 구분된 섹션들이 뒤섞여 검색 품질이 떨어집니다
- **코드 저장소**: Python 파일을 분할했는데, 함수 정의가 반으로 잘려서 코드 검색이 무의미해집니다
- **다양한 주제 문서**: 한 문서 안에 여러 주제가 섞여 있는데, 주제가 바뀌는 지점을 자동으로 감지하고 싶습니다

고급 텍스트 분할기는 이런 문제를 해결합니다. 문서의 **구조적 단서**(헤더, 태그, 문법)나 **의미적 단서**(임베딩 유사도)를 활용해, 정보의 맥락을 보존하면서 분할하죠.

## 핵심 개념

### 개념 1: MarkdownHeaderTextSplitter — 마크다운 계층 구조 보존

> 💡 **비유**: 책의 목차를 떠올려 보세요. "1장 > 1.1절 > 1.1.1항"처럼 계층 구조가 있잖아요? `MarkdownHeaderTextSplitter`는 마크다운의 `#`, `##`, `###` 헤더를 이 목차처럼 읽어서, 각 섹션을 독립적인 청크로 만들면서도 "이 내용은 1장 > 1.1절에 속한다"는 위치 정보를 메타데이터로 보존합니다.

`MarkdownHeaderTextSplitter`는 마크다운 문서를 **헤더 기준으로 분할**하고, 각 청크에 **헤더 계층 정보를 메타데이터**로 추가합니다. 글자 수나 토큰 수가 아니라, 문서의 논리적 구조를 기준으로 자르는 거죠.

```python
from langchain_text_splitters import MarkdownHeaderTextSplitter

# 분할 기준이 될 헤더와 메타데이터 키 지정
headers_to_split_on = [
    ("#", "Header 1"),    # h1 → "Header 1" 메타데이터 키
    ("##", "Header 2"),   # h2 → "Header 2" 메타데이터 키
    ("###", "Header 3"),  # h3 → "Header 3" 메타데이터 키
]

splitter = MarkdownHeaderTextSplitter(
    headers_to_split_on=headers_to_split_on,
    strip_headers=False,  # 헤더를 청크 내용에 포함할지 여부
)

# 예시 마크다운 문서
markdown_doc = """# LangChain 가이드

LangChain은 LLM 기반 애플리케이션 프레임워크입니다.

## 설치 방법

pip install langchain으로 설치할 수 있습니다.

### 의존성 관리

python-dotenv와 openai 패키지도 함께 설치하세요.

## LCEL 소개

LCEL은 파이프 연산자로 체인을 구성합니다.

### 파이프 연산자

`|` 연산자로 컴포넌트를 연결합니다.
"""

chunks = splitter.split_text(markdown_doc)

for chunk in chunks:
    print(f"내용: {chunk.page_content[:50]}...")
    print(f"메타데이터: {chunk.metadata}")
    print("---")
```

실행 결과를 보면, 각 청크의 메타데이터에 해당 섹션이 속한 상위 헤더 정보가 포함됩니다:

```
내용: # LangChain 가이드
LangChain은 LLM 기반 애플리케이션 ...
메타데이터: {'Header 1': 'LangChain 가이드'}
---
내용: ## 설치 방법
pip install langchain으로 설치할 수 ...
메타데이터: {'Header 1': 'LangChain 가이드', 'Header 2': '설치 방법'}
---
내용: ### 의존성 관리
python-dotenv와 openai 패키지도 ...
메타데이터: {'Header 1': 'LangChain 가이드', 'Header 2': '설치 방법', 'Header 3': '의존성 관리'}
---
```

이 메타데이터가 왜 중요할까요? RAG에서 검색할 때 "설치 방법"이라는 쿼리가 들어오면, 메타데이터의 `Header 2` 값을 활용해 더 정확한 필터링이 가능하기 때문입니다.

**2단계 분할 패턴**: 헤더로 분할한 뒤, 각 섹션이 여전히 너무 크다면 `RecursiveCharacterTextSplitter`와 조합할 수 있습니다.

```python
from langchain_text_splitters import (
    MarkdownHeaderTextSplitter,
    RecursiveCharacterTextSplitter,
)

# 1단계: 헤더 기준 분할
md_splitter = MarkdownHeaderTextSplitter(
    headers_to_split_on=headers_to_split_on,
    strip_headers=False,
)
md_chunks = md_splitter.split_text(markdown_doc)

# 2단계: 큰 섹션을 추가로 분할 (메타데이터 유지)
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,
    chunk_overlap=50,
)
final_chunks = text_splitter.split_documents(md_chunks)
# split_documents를 사용하면 원래 메타데이터가 보존됩니다!
```

### 개념 2: HTMLHeaderTextSplitter — HTML 문서의 구조적 분할

> 💡 **비유**: 신문을 생각해 보세요. 큰 헤드라인(1면 톱), 중간 제목(섹션 헤더), 소제목이 있죠. `HTMLHeaderTextSplitter`는 HTML의 `<h1>`, `<h2>`, `<h3>` 태그를 이런 신문의 제목 계층처럼 인식해서, 각 섹션을 의미 단위로 분리합니다.

웹에서 크롤링한 HTML 데이터를 다룰 때 특히 유용합니다. `HTMLHeaderTextSplitter`는 HTML 헤더 태그를 기준으로 문서를 분할하고, 각 청크에 해당 헤더 정보를 메타데이터로 보존합니다.

```python
from langchain_text_splitters import HTMLHeaderTextSplitter

# 분할 기준 헤더 태그 지정
headers_to_split_on = [
    ("h1", "Header 1"),
    ("h2", "Header 2"),
    ("h3", "Header 3"),
]

splitter = HTMLHeaderTextSplitter(headers_to_split_on=headers_to_split_on)

# HTML 문서 또는 URL에서 분할
html_string = """
<!DOCTYPE html>
<html>
<body>
    <h1>LangChain 튜토리얼</h1>
    <p>LangChain을 활용한 AI 애플리케이션 개발 가이드입니다.</p>

    <h2>시작하기</h2>
    <p>먼저 환경을 설정합니다.</p>
    <p>Python 3.10 이상이 필요합니다.</p>

    <h3>패키지 설치</h3>
    <p>pip install langchain langchain-openai</p>

    <h2>핵심 개념</h2>
    <p>LangChain의 핵심은 LCEL입니다.</p>
    <p>파이프 연산자로 체인을 구성합니다.</p>
</body>
</html>
"""

chunks = splitter.split_text(html_string)

for chunk in chunks:
    print(f"내용: {chunk.page_content}")
    print(f"메타데이터: {chunk.metadata}")
    print("---")
```

```
내용: LangChain을 활용한 AI 애플리케이션 개발 가이드입니다.
메타데이터: {'Header 1': 'LangChain 튜토리얼'}
---
내용: 먼저 환경을 설정합니다. Python 3.10 이상이 필요합니다.
메타데이터: {'Header 1': 'LangChain 튜토리얼', 'Header 2': '시작하기'}
---
내용: pip install langchain langchain-openai
메타데이터: {'Header 1': 'LangChain 튜토리얼', 'Header 2': '시작하기', 'Header 3': '패키지 설치'}
---
내용: LangChain의 핵심은 LCEL입니다. 파이프 연산자로 체인을 구성합니다.
메타데이터: {'Header 1': 'LangChain 튜토리얼', 'Header 2': '핵심 개념'}
---
```

URL에서 직접 HTML을 가져와 분할할 수도 있습니다:

```python
# URL에서 직접 로드하여 분할
url = "https://python.langchain.com/docs/get_started/introduction"
chunks = splitter.split_text_from_url(url)
```

`HTMLHeaderTextSplitter` 역시 `RecursiveCharacterTextSplitter`와 조합하는 2단계 분할 패턴을 사용할 수 있습니다. [6.2: 다양한 소스 로더](ch06/session_02.md)에서 배운 `WebBaseLoader`로 웹 페이지를 가져온 뒤, 이 분할기로 구조를 보존하며 청크를 만들면 웹 데이터 기반 RAG의 품질이 크게 향상됩니다.

### 개념 3: 코드 분할 — Language enum과 from_language()

> 💡 **비유**: 소설을 문장 단위로 자르는 것과, 프로그래밍 코드를 자르는 것은 완전히 다릅니다. 소설에서 마침표는 문장의 끝이지만, 코드에서는 `class`, `def`, `function` 같은 키워드가 논리적 단위의 시작이죠. `from_language()`는 각 프로그래밍 언어의 "문법 사전"을 내장하고 있어서, 코드의 논리적 구조를 존중하며 분할합니다.

LangChain은 `Language` enum을 통해 **25개 이상의 프로그래밍 언어**를 지원합니다. `RecursiveCharacterTextSplitter.from_language()`를 사용하면, 해당 언어의 문법 구조(클래스, 함수, 메서드 등)를 인식하는 분할기가 자동으로 구성됩니다.

```python
from langchain_text_splitters import Language, RecursiveCharacterTextSplitter

# 지원되는 전체 언어 목록 확인
print([lang.value for lang in Language])
# ['cpp', 'go', 'java', 'kotlin', 'js', 'ts', 'php', 'proto',
#  'python', 'rst', 'ruby', 'rust', 'scala', 'swift', 'markdown',
#  'latex', 'html', 'sol', 'csharp', 'cobol', 'c', 'lua', 'perl',
#  'haskell', 'elixir']
```

각 언어별로 어떤 구분자를 사용하는지 확인할 수 있습니다:

```python
# Python의 분할 구분자 확인
python_seps = RecursiveCharacterTextSplitter.get_separators_for_language(
    Language.PYTHON
)
print(python_seps)
# ['\nclass ', '\ndef ', '\n\ndef ', '\n\n', '\n', ' ', '']
# → 클래스 → 함수 → 빈 줄 → 줄바꿈 → 공백 순으로 분할 시도

# JavaScript의 분할 구분자 확인
js_seps = RecursiveCharacterTextSplitter.get_separators_for_language(
    Language.JS
)
print(js_seps)
# ['\nfunction ', '\nconst ', '\nlet ', '\nvar ', '\nclass ',
#  '\nif ', '\nfor ', '\nwhile ', '\nswitch ', '\ncase ',
#  '\ndefault ', '\n\n', '\n', ' ', '']
```

실제 Python 코드를 분할해 봅시다:

```python
python_code = """
import os
from typing import List

class DocumentProcessor:
    \"\"\"문서 처리를 담당하는 클래스\"\"\"

    def __init__(self, chunk_size: int = 1000):
        self.chunk_size = chunk_size
        self.processed_count = 0

    def load_document(self, file_path: str) -> str:
        \"\"\"파일에서 문서를 로드합니다.\"\"\"
        with open(file_path, 'r', encoding='utf-8') as f:
            content = f.read()
        self.processed_count += 1
        return content

    def split_text(self, text: str) -> List[str]:
        \"\"\"텍스트를 청크로 분할합니다.\"\"\"
        chunks = []
        for i in range(0, len(text), self.chunk_size):
            chunks.append(text[i:i + self.chunk_size])
        return chunks


def create_processor(chunk_size: int = 500) -> DocumentProcessor:
    \"\"\"DocumentProcessor 인스턴스를 생성합니다.\"\"\"
    return DocumentProcessor(chunk_size=chunk_size)


def main():
    processor = create_processor()
    content = processor.load_document("example.txt")
    chunks = processor.split_text(content)
    print(f"총 {len(chunks)}개 청크 생성")
"""

# Python 코드 전용 분할기 생성
python_splitter = RecursiveCharacterTextSplitter.from_language(
    language=Language.PYTHON,
    chunk_size=200,      # 예시를 위해 작은 크기 사용
    chunk_overlap=20,
)

chunks = python_splitter.split_text(python_code)

for i, chunk in enumerate(chunks):
    print(f"=== 청크 {i + 1} ===")
    print(chunk)
    print()
```

`from_language(Language.PYTHON)`을 사용하면, 클래스와 함수 정의가 가능한 한 하나의 청크에 온전히 포함됩니다. 일반 `RecursiveCharacterTextSplitter`를 사용했다면 `def load_document` 함수가 중간에 잘릴 수 있지만, 언어 인식 분할기는 `\ndef ` 구분자를 우선적으로 사용해 함수 경계에서 분할하려고 시도합니다.

### 개념 4: SemanticChunker — 의미 기반 분할

> 📌 **선수 지식 안내**: SemanticChunker는 **임베딩(embedding)**과 **코사인 유사도(cosine similarity)** 개념을 활용합니다. 이 개념들은 [7.1: 임베딩의 이해](ch07/session_01.md)에서 자세히 다룹니다. 여기서는 **개념적·직관적 이해만으로 충분**하며, Ch7 학습 후 이 섹션을 다시 방문하면 더 깊이 이해할 수 있습니다.

> 💡 **비유**: 친구가 여행 이야기를 할 때를 떠올려 보세요. "파리 에펠탑 이야기"를 하다가 갑자기 "도쿄 라멘 이야기"로 넘어가면, 우리는 자연스럽게 "아, 주제가 바뀌었구나"라고 느끼죠. `SemanticChunker`는 바로 이렇게 **의미가 전환되는 지점**을 감지해서, 같은 주제끼리 묶어주는 분할기입니다.

`SemanticChunker`는 앞서 배운 모든 분할기와 근본적으로 다릅니다. 글자 수, 헤더, 문법 같은 **형식적 단서**가 아니라, **텍스트의 의미적 유사도**를 기준으로 분할합니다. `langchain_experimental` 패키지에 포함되어 있습니다.

**임베딩이란? — 직관적으로 이해하기**

임베딩은 텍스트를 **숫자 좌표로 변환**하는 것이라고 생각하면 됩니다. 지도에서 서울과 부산의 좌표가 있듯이, 텍스트도 의미 공간에서 좌표를 가집니다.

- "강아지"와 "puppy"는 비슷한 좌표에 놓입니다 (의미가 비슷하니까)
- "강아지"와 "자동차"는 먼 좌표에 놓입니다 (의미가 다르니까)
- 두 좌표 사이의 거리가 가까우면 "의미적으로 유사하다"고 판단합니다

이 좌표 간 거리를 측정하는 방법 중 하나가 **코사인 유사도**인데, 수학적 세부사항은 [7.1: 임베딩의 이해](ch07/session_01.md)에서 다룹니다. 지금은 "텍스트 간 의미적 거리를 숫자로 측정할 수 있다"는 점만 기억하면 충분합니다.

**동작 원리**:
1. 텍스트를 문장 단위로 분리
2. 인접한 문장들을 그룹으로 묶어 **"의미 좌표"를 계산** (= 임베딩 생성)
3. 연속된 그룹 간 **"의미적 거리"를 측정** (= 코사인 유사도 비교)
4. 거리가 급격히 벌어지는 지점 = **"주제가 바뀌는 지점"**에서 분할

```python
from langchain_experimental.text_splitter import SemanticChunker
from langchain_openai import OpenAIEmbeddings

# 임베딩 모델 기반 시맨틱 청커 생성
semantic_chunker = SemanticChunker(
    embeddings=OpenAIEmbeddings(),
    breakpoint_threshold_type="percentile",  # 분할 임계값 전략
    breakpoint_threshold_amount=95,          # 상위 5%의 거리에서 분할
)

text = """
인공지능의 역사는 1950년대로 거슬러 올라갑니다. 앨런 튜링은 1950년에
"기계가 생각할 수 있는가?"라는 질문을 던졌습니다. 이후 1956년 다트머스
회의에서 '인공지능'이라는 용어가 처음 사용되었습니다.

딥러닝은 인공신경망의 한 분야입니다. 여러 층의 뉴런을 쌓아 복잡한
패턴을 학습합니다. 2012년 AlexNet이 ImageNet 대회에서 우승하면서
딥러닝 시대가 본격적으로 열렸습니다.

한편, 자연어 처리 분야에서는 2017년 Transformer 아키텍처가 등장했습니다.
Attention 메커니즘을 기반으로 한 이 모델은 기계 번역의 성능을 획기적으로
향상시켰습니다. 이후 BERT, GPT 등의 대규모 언어 모델로 발전했습니다.

최근 RAG(Retrieval-Augmented Generation) 기법이 주목받고 있습니다.
LLM의 환각 문제를 해결하기 위해, 외부 지식을 검색하여 응답에 활용합니다.
LangChain은 이러한 RAG 파이프라인을 쉽게 구축할 수 있게 도와줍니다.
"""

chunks = semantic_chunker.split_text(text)

for i, chunk in enumerate(chunks):
    print(f"=== 시맨틱 청크 {i + 1} ===")
    print(chunk[:100] + "..." if len(chunk) > 100 else chunk)
    print()
```

위 예시에서 SemanticChunker는 "AI 역사", "딥러닝", "자연어 처리", "RAG" 등 **주제가 전환되는 지점**을 자동으로 감지하여 분할합니다. 글자 수가 아니라 의미의 흐름을 기준으로 자르기 때문에, 같은 주제의 문장들이 하나의 청크에 자연스럽게 묶입니다.

**4가지 분할 전략(breakpoint_threshold_type)**:

| 전략 | 기본값 | 설명 |
|------|--------|------|
| `percentile` | 95 | 거리 분포의 상위 N% 지점에서 분할 (가장 일반적) |
| `standard_deviation` | 3 | 평균 + N×표준편차 초과 지점에서 분할 |
| `interquartile` | 1.5 | IQR의 N배를 초과하는 지점에서 분할 |
| `gradient` | 95 | 거리 변화율(기울기)의 상위 N% 지점에서 분할 |

각 전략은 "의미적 거리가 얼마나 벌어져야 분할할 것인가"를 다른 통계적 기준으로 판단합니다. 처음에는 `percentile`로 시작하고, 결과를 보며 다른 전략을 실험해 보세요.

```python
# 다양한 전략 비교
strategies = [
    ("percentile", 90),
    ("standard_deviation", 2),
    ("interquartile", 1.5),
    ("gradient", 90),
]

for strategy, threshold in strategies:
    chunker = SemanticChunker(
        embeddings=OpenAIEmbeddings(),
        breakpoint_threshold_type=strategy,
        breakpoint_threshold_amount=threshold,
    )
    chunks = chunker.split_text(text)
    print(f"{strategy} (임계값={threshold}): {len(chunks)}개 청크")
```

> ⚠️ **흔한 오해**: "SemanticChunker가 항상 최고의 결과를 준다"고 생각하기 쉽지만, 실제로는 그렇지 않습니다. 짧은 문서나 주제가 균일한 문서에서는 오히려 `RecursiveCharacterTextSplitter`가 더 안정적인 결과를 줄 수 있습니다. SemanticChunker는 **여러 주제가 섞인 긴 문서**에서 진가를 발휘합니다.

> 📚 **학습 로드맵**: SemanticChunker의 임베딩 원리가 궁금하다면, [7.1: 임베딩의 이해](ch07/session_01.md)를 학습한 뒤 이 섹션을 다시 방문해 보세요. 벡터 공간에서 코사인 유사도가 어떻게 계산되는지 이해하면, `breakpoint_threshold_type`의 각 전략이 왜 다른 결과를 만드는지 훨씬 명확해집니다.

## 실습: 직접 해보기

실무에서 자주 만나는 시나리오를 종합적으로 다뤄봅시다. 기술 문서(마크다운), 웹 페이지(HTML), 소스 코드(Python)를 각각 적합한 분할기로 처리하는 **멀티포맷 문서 처리 파이프라인**을 구축합니다.

```python
"""
멀티포맷 문서 처리 파이프라인
- 마크다운, HTML, Python 코드를 각 형식에 최적화된 분할기로 처리
"""

from langchain_text_splitters import (
    MarkdownHeaderTextSplitter,
    HTMLHeaderTextSplitter,
    RecursiveCharacterTextSplitter,
    Language,
)

# ============================================================
# 1. 마크다운 기술 문서 분할
# ============================================================

markdown_doc = """# RAG 시스템 구축 가이드

RAG는 검색 증강 생성의 약자입니다.

## 1단계: 문서 로딩

다양한 형식의 문서를 로드합니다. TextLoader, PyPDFLoader,
CSVLoader 등을 활용하세요. 대용량 파일은 lazy_load()를 사용하면
메모리를 절약할 수 있습니다.

### 로더 선택 기준

파일 형식에 따라 적절한 로더를 선택하는 것이 중요합니다.
텍스트 파일은 TextLoader, PDF는 PyPDFLoader를 사용합니다.
구조화된 데이터는 CSVLoader나 JSONLoader가 적합합니다.

## 2단계: 텍스트 분할

로드된 문서를 적절한 크기로 분할합니다. chunk_size와
chunk_overlap을 조정하여 최적의 분할 결과를 얻으세요.

### 분할 전략 선택

일반 텍스트는 RecursiveCharacterTextSplitter,
마크다운은 MarkdownHeaderTextSplitter,
코드는 from_language()를 사용하세요.

## 3단계: 임베딩과 저장

분할된 청크를 벡터로 변환하여 벡터 스토어에 저장합니다.
"""

# 마크다운 헤더 기반 분할
md_headers = [
    ("#", "제목"),
    ("##", "단계"),
    ("###", "세부항목"),
]

md_splitter = MarkdownHeaderTextSplitter(
    headers_to_split_on=md_headers,
    strip_headers=False,  # 헤더를 내용에 포함
)
md_chunks = md_splitter.split_text(markdown_doc)

# 큰 섹션은 추가 분할
char_splitter = RecursiveCharacterTextSplitter(
    chunk_size=200,
    chunk_overlap=30,
)
md_final = char_splitter.split_documents(md_chunks)

print("=== 마크다운 분할 결과 ===")
for i, doc in enumerate(md_final):
    print(f"\n[청크 {i + 1}]")
    print(f"  메타데이터: {doc.metadata}")
    print(f"  내용: {doc.page_content[:80]}...")

# ============================================================
# 2. HTML 웹 페이지 분할
# ============================================================

html_content = """
<html>
<body>
    <h1>Python 패키지 관리</h1>
    <p>Python 프로젝트의 의존성을 효과적으로 관리하는 방법을 알아봅니다.</p>

    <h2>pip 사용법</h2>
    <p>pip는 Python의 기본 패키지 관리자입니다.</p>
    <p>pip install package_name으로 패키지를 설치합니다.</p>

    <h3>requirements.txt</h3>
    <p>프로젝트 의존성을 파일로 관리할 수 있습니다.</p>
    <p>pip freeze > requirements.txt로 현재 환경을 저장합니다.</p>

    <h2>가상환경</h2>
    <p>프로젝트별로 독립된 Python 환경을 만들 수 있습니다.</p>
    <p>python -m venv myenv 명령으로 가상환경을 생성합니다.</p>
</body>
</html>
"""

html_headers = [
    ("h1", "Header 1"),
    ("h2", "Header 2"),
    ("h3", "Header 3"),
]

html_splitter = HTMLHeaderTextSplitter(headers_to_split_on=html_headers)
html_chunks = html_splitter.split_text(html_content)

print("\n\n=== HTML 분할 결과 ===")
for i, doc in enumerate(html_chunks):
    print(f"\n[청크 {i + 1}]")
    print(f"  메타데이터: {doc.metadata}")
    print(f"  내용: {doc.page_content[:80]}...")

# ============================================================
# 3. Python 소스 코드 분할
# ============================================================

python_source = '''
import os
from dataclasses import dataclass
from typing import Optional


@dataclass
class Config:
    """애플리케이션 설정을 담는 클래스"""
    api_key: str
    model_name: str = "gpt-4o"
    temperature: float = 0.7
    max_tokens: Optional[int] = None


class RAGPipeline:
    """RAG 파이프라인을 구현하는 메인 클래스"""

    def __init__(self, config: Config):
        self.config = config
        self.documents = []
        self.chunks = []

    def load_documents(self, directory: str) -> list:
        """디렉토리에서 모든 문서를 로드합니다."""
        loaded = []
        for filename in os.listdir(directory):
            filepath = os.path.join(directory, filename)
            if filepath.endswith(".txt"):
                with open(filepath, "r") as f:
                    loaded.append(f.read())
        self.documents = loaded
        return loaded

    def split_documents(self, chunk_size: int = 500) -> list:
        """로드된 문서를 청크로 분할합니다."""
        self.chunks = []
        for doc in self.documents:
            for i in range(0, len(doc), chunk_size):
                self.chunks.append(doc[i:i + chunk_size])
        return self.chunks

    def search(self, query: str, top_k: int = 3) -> list:
        """쿼리와 유사한 청크를 검색합니다."""
        # 간단한 키워드 매칭 (실제로는 벡터 검색 사용)
        results = [c for c in self.chunks if query.lower() in c.lower()]
        return results[:top_k]


def main():
    config = Config(api_key=os.getenv("OPENAI_API_KEY"))
    pipeline = RAGPipeline(config)
    pipeline.load_documents("./docs")
    pipeline.split_documents()
    results = pipeline.search("RAG")
    for r in results:
        print(r)
'''

# Python 코드 전용 분할기
python_splitter = RecursiveCharacterTextSplitter.from_language(
    language=Language.PYTHON,
    chunk_size=300,    # 코드는 더 작은 청크로
    chunk_overlap=30,
)
code_chunks = python_splitter.split_text(python_source)

print("\n\n=== Python 코드 분할 결과 ===")
for i, chunk in enumerate(code_chunks):
    # 각 청크의 첫 줄과 길이 표시
    first_line = chunk.strip().split('\n')[0]
    print(f"\n[청크 {i + 1}] (길이: {len(chunk)}자)")
    print(f"  시작: {first_line}")

# ============================================================
# 4. 분할기 선택 가이드 출력
# ============================================================

print("\n\n=== 분할기 선택 가이드 ===")
guide = {
    "마크다운 문서": "MarkdownHeaderTextSplitter → RecursiveCharacterTextSplitter",
    "HTML 웹 페이지": "HTMLHeaderTextSplitter → RecursiveCharacterTextSplitter",
    "소스 코드": "RecursiveCharacterTextSplitter.from_language()",
    "일반 텍스트": "RecursiveCharacterTextSplitter",
    "다주제 긴 문서": "SemanticChunker",
}

for doc_type, splitter in guide.items():
    print(f"  {doc_type:20s} → {splitter}")
```

## 더 깊이 알아보기

### 시맨틱 청킹의 탄생 배경

이 세션에서 다루는 분할 전략들은 [6.3](ch06/session_03.md)에서 소개한 Greg Kamradt의 "5 Levels of Text Splitting" 프레임워크 중 **Level 3(문서 구조 인식)과 Level 4(시맨틱 청킹)**에 해당합니다. Level 1-2의 기초적인 분할 전략은 6.3에서 이미 다루었으므로, 여기서는 구조와 의미를 활용하는 고급 전략에 집중합니다.

Kamradt의 핵심 인사이트는 간단했습니다: "첫 문장의 임베딩을 구한 뒤, 다음 문장들과 비교하면서 거리가 크게 벌어지는 지점을 새로운 의미 단위의 시작으로 보자." 이 아이디어가 LangChain 팀에 의해 `SemanticChunker`로 구현된 것이죠.

한편, Level 5인 "추론 기반 청킹"(Agentic Chunking)은 LLM을 호출해서 "이 문장은 이전 청크에 속하는가, 새 청크를 시작해야 하는가?"를 판단하는 방식인데, 비용과 속도 때문에 아직 실무에서 널리 쓰이지는 않습니다. 하지만 모델 비용이 낮아지면서 점차 현실적인 옵션이 되어가고 있습니다.

### 구조 인식 분할의 원리

`MarkdownHeaderTextSplitter`나 `HTMLHeaderTextSplitter`가 내부적으로 하는 일은, 정규표현식이나 HTML 파서(BeautifulSoup)로 헤더를 감지하고, **헤더 간의 내용을 하나의 논리 단위로 묶는 것**입니다. 이때 헤더의 계층(`#` > `##` > `###`)을 추적하여, 하위 섹션이 어떤 상위 섹션에 속하는지 메타데이터로 기록합니다.

이는 단순히 "잘 자르는 것"을 넘어, **검색 시 맥락 정보를 활용할 수 있게 해주는** 강력한 패턴입니다. 예를 들어 "설치 방법"이라는 쿼리가 들어왔을 때, 메타데이터의 `Header 2 == "설치 방법"`인 청크들을 우선 반환하거나, 검색 결과에 "이 내용은 'LangChain 가이드 > 설치 방법' 섹션에 있습니다"라는 출처 정보를 자동으로 제공할 수 있습니다.

## 흔한 오해와 팁

> ⚠️ **흔한 오해**: "고급 분할기를 쓰면 항상 더 좋은 결과를 얻는다"고 생각하기 쉽습니다. 하지만 실제로는 **데이터의 특성에 맞는 분할기**를 선택하는 것이 핵심입니다. 헤더가 없는 마크다운에 `MarkdownHeaderTextSplitter`를 쓰면 전체가 하나의 청크가 되어버리고, 짧은 문서에 `SemanticChunker`를 쓰면 불필요하게 잘게 쪼개질 수 있습니다. 가장 중요한 건 데이터를 먼저 살펴보는 것입니다.

> 💡 **알고 계셨나요?**: `Language` enum은 프로그래밍 언어뿐 아니라 `MARKDOWN`, `LATEX`, `HTML`, `RST` 같은 마크업 언어도 지원합니다. 그래서 `from_language(Language.MARKDOWN)`을 사용하면 `MarkdownHeaderTextSplitter`와는 다른 방식으로 마크다운을 분할할 수 있는데요 — 전자는 헤더, 코드 블록, 리스트 등의 마크다운 문법 요소를 구분자로 사용하고, 후자는 헤더 계층을 메타데이터로 보존합니다. 목적에 따라 선택하세요!

> 🔥 **실무 팁**: 실전에서 가장 많이 쓰이는 패턴은 **2단계 분할**입니다. 1단계로 구조 인식 분할기(`MarkdownHeaderTextSplitter`, `HTMLHeaderTextSplitter`)로 논리적 섹션을 나누고, 2단계로 `RecursiveCharacterTextSplitter`의 `split_documents()`를 사용해 큰 섹션을 추가 분할합니다. 이때 `split_documents()`를 쓰면 **1단계에서 생성된 메타데이터가 자동으로 보존**되므로, 반드시 `split_text()` 대신 `split_documents()`를 사용하세요.

> 🔥 **실무 팁**: `SemanticChunker`는 임베딩 API를 호출하므로 **비용과 속도**에 주의하세요. 대량의 문서를 처리할 때는 먼저 구조 기반 분할로 대략적인 섹션을 나누고, 정말 필요한 경우에만 시맨틱 청킹을 적용하는 것이 현실적입니다. 또한 `breakpoint_threshold_type`은 `percentile`로 시작한 뒤, 결과를 보며 `standard_deviation`이나 `gradient`를 실험해 보세요.

## 핵심 정리

| 개념 | 설명 |
|------|------|
| `MarkdownHeaderTextSplitter` | 마크다운 헤더(`#`, `##`, `###`) 기준으로 분할하고, 헤더 계층을 메타데이터로 보존 |
| `HTMLHeaderTextSplitter` | HTML 헤더 태그(`<h1>`~`<h6>`) 기준으로 분할, URL에서 직접 로드 가능 |
| `Language` enum | 25개 이상의 프로그래밍/마크업 언어를 지원하는 열거형 |
| `from_language()` | 지정 언어의 문법 구분자를 자동 설정한 `RecursiveCharacterTextSplitter` 반환 |
| `get_separators_for_language()` | 특정 언어의 분할 구분자 목록을 확인하는 클래스 메서드 |
| `SemanticChunker` | 임베딩 유사도 기반으로 의미 전환 지점에서 분할 (`langchain_experimental`). 임베딩 기초는 Ch7에서 상세 학습 |
| `breakpoint_threshold_type` | 시맨틱 분할의 임계값 전략: `percentile`, `standard_deviation`, `interquartile`, `gradient` |
| 2단계 분할 패턴 | 구조 인식 분할 → `RecursiveCharacterTextSplitter.split_documents()`로 메타데이터 보존하며 추가 분할 |
