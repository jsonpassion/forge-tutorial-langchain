# 고급 프롬프트 패턴

> MessagesPlaceholder, 조건부 프롬프트, 프롬프트 합성, 멀티모달 프롬프트까지 — ChatPromptTemplate의 숨겨진 능력을 파헤칩니다.

## 개요

이 섹션에서는 [Session 3.1: ChatPromptTemplate 기초](./01-chatprompttemplate-기초.md)에서 배운 기본기를 넘어, 실전에서 자주 쓰이는 고급 프롬프트 패턴 네 가지를 다룹니다. 대화 기록을 동적으로 삽입하는 `MessagesPlaceholder`, 입력 조건에 따라 다른 프롬프트를 선택하는 조건부 라우팅, 여러 프롬프트를 레고처럼 합성하는 기법, 그리고 이미지와 텍스트를 함께 처리하는 멀티모달 프롬프트를 학습합니다.

**선수 지식**: `ChatPromptTemplate.from_messages()`, `SystemMessage`/`HumanMessage`/`AIMessage`의 역할 차이, LCEL 파이프 연산자(`|`) 기본 사용법 (Session 3.1에서 학습)
**학습 목표**:
- `MessagesPlaceholder`를 사용하여 대화 기록이나 동적 메시지 목록을 프롬프트에 삽입할 수 있다
- `RunnableBranch`를 활용해 입력 조건에 따라 다른 프롬프트를 라우팅할 수 있다
- `+` 연산자와 `append`/`extend`를 사용해 프롬프트를 합성할 수 있다
- 이미지를 포함한 멀티모달 프롬프트를 구성할 수 있다

## 왜 알아야 할까?

실제 LLM 애플리케이션을 만들어 보면, 단순히 "시스템 메시지 + 사용자 메시지"만으로는 부족한 순간이 금방 찾아옵니다. 챗봇은 이전 대화를 기억해야 하고, 고객 문의 시스템은 문의 유형에 따라 다른 프롬프트를 사용해야 하며, 이미지 분석 서비스는 텍스트와 이미지를 함께 처리해야 하죠.

Session 3.1에서 배운 `ChatPromptTemplate`은 사실 빙산의 일각이었습니다. 오늘 배울 패턴들은 프로덕션 수준의 LangChain 애플리케이션에서 거의 반드시 사용되는 것들이에요. 특히 `MessagesPlaceholder`는 Chapter 10에서 다룰 메모리 시스템의 핵심 기반이 되고, 조건부 프롬프트는 Chapter 5의 LCEL 심화 내용과 직결됩니다.

## 핵심 개념

### 개념 1: MessagesPlaceholder — 대화 기록의 "빈 칸"

> 💡 **비유**: 대학 강의 노트를 생각해 보세요. 매 수업마다 새 노트를 쓰지만, 앞부분에 "지난 수업 복습" 칸을 비워 둡니다. 수업이 진행될수록 그 빈 칸에 이전 수업 내용이 차곡차곡 채워지죠. `MessagesPlaceholder`는 바로 이 "빈 칸"입니다 — 프롬프트에 메시지 목록이 들어갈 자리를 미리 예약해 두는 거예요.

`MessagesPlaceholder`는 `ChatPromptTemplate` 안에서 **메시지 리스트가 동적으로 삽입될 위치**를 지정합니다. 앞서 배운 `{variable}`이 단일 문자열을 받는다면, `MessagesPlaceholder`는 여러 개의 메시지 객체 리스트를 통째로 받을 수 있습니다.

```python
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_core.messages import HumanMessage, AIMessage

# MessagesPlaceholder로 대화 기록 자리 만들기
prompt = ChatPromptTemplate.from_messages([
    ("system", "당신은 친절한 AI 비서입니다."),
    MessagesPlaceholder("chat_history"),  # 여기에 대화 기록이 삽입됨
    ("human", "{question}"),
])

# 이전 대화 기록
history = [
    HumanMessage(content="파이썬이 뭐야?"),
    AIMessage(content="파이썬은 범용 프로그래밍 언어입니다."),
]

# 프롬프트 생성
result = prompt.invoke({
    "chat_history": history,
    "question": "그럼 어디에 주로 쓰여?"
})

print(result.to_messages())
# 출력:
# [SystemMessage(content='당신은 친절한 AI 비서입니다.'),
#  HumanMessage(content='파이썬이 뭐야?'),
#  AIMessage(content='파이썬은 범용 프로그래밍 언어입니다.'),
#  HumanMessage(content='그럼 어디에 주로 쓰여?')]
```

핵심은 `chat_history`라는 변수에 메시지 리스트를 전달하면, 그것이 시스템 메시지와 현재 질문 사이에 자연스럽게 삽입된다는 점이에요.

#### optional 파라미터: 대화 기록이 없을 때

첫 대화에서는 기록이 없겠죠? 이때 `optional=True`를 설정하면 `chat_history`를 제공하지 않아도 에러 없이 빈 리스트로 처리됩니다.

```python
# optional=True: 대화 기록이 없어도 에러 없음
prompt = ChatPromptTemplate.from_messages([
    ("system", "당신은 친절한 AI 비서입니다."),
    MessagesPlaceholder("chat_history", optional=True),  # 없으면 빈 리스트
    ("human", "{question}"),
])

# 첫 대화 — chat_history 없이 호출 가능
result = prompt.invoke({"question": "안녕하세요!"})
print(result.to_messages())
# 출력:
# [SystemMessage(content='당신은 친절한 AI 비서입니다.'),
#  HumanMessage(content='안녕하세요!')]
```

#### 튜플 단축 문법

`MessagesPlaceholder`를 매번 임포트하기 귀찮다면, 튜플 단축 문법을 사용할 수도 있습니다.

```python
# 아래 두 줄은 동일한 동작
MessagesPlaceholder("chat_history", optional=True)
("placeholder", "{chat_history}")  # 튜플 단축 문법
```

> ⚠️ **흔한 오해**: `MessagesPlaceholder`에 문자열 하나를 넘기면 에러가 납니다. 반드시 **메시지 객체의 리스트**를 전달해야 해요. `{"chat_history": "이전 대화"}` (X) → `{"chat_history": [HumanMessage(content="이전 대화")]}` (O)

### 개념 2: 조건부 프롬프트 — 상황에 맞는 프롬프트 자동 선택

> 💡 **비유**: 병원의 접수 데스크를 떠올려 보세요. 환자가 오면 증상에 따라 내과, 외과, 피부과 중 적절한 곳으로 안내하죠. 조건부 프롬프트도 마찬가지예요 — 사용자 입력의 특성을 파악해서, 가장 적합한 프롬프트로 자동 라우팅합니다.

실무에서는 하나의 프롬프트로 모든 상황을 처리하기 어렵습니다. 코드 관련 질문에는 코드 전문가 프롬프트를, 일반 대화에는 친근한 챗봇 프롬프트를 사용하는 것이 훨씬 효과적이죠. LangChain의 `RunnableBranch`를 사용하면 이런 조건부 라우팅을 선언적으로 구현할 수 있습니다.

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import RunnableBranch, RunnablePassthrough
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI

# 목적별 프롬프트 정의
code_prompt = ChatPromptTemplate.from_messages([
    ("system", "당신은 시니어 소프트웨어 엔지니어입니다. 코드 예제와 함께 설명하세요."),
    ("human", "{input}"),
])

general_prompt = ChatPromptTemplate.from_messages([
    ("system", "당신은 친절하고 박식한 AI 비서입니다."),
    ("human", "{input}"),
])

creative_prompt = ChatPromptTemplate.from_messages([
    ("system", "당신은 창의적인 작가입니다. 비유와 스토리텔링을 활용하세요."),
    ("human", "{input}"),
])

# 조건 판별 함수
def is_code_question(input_dict: dict) -> bool:
    """코드 관련 키워드가 포함되어 있는지 확인"""
    keywords = ["코드", "프로그래밍", "함수", "버그", "에러", "python", "java"]
    return any(kw in input_dict["input"].lower() for kw in keywords)

def is_creative_request(input_dict: dict) -> bool:
    """창작 관련 키워드가 포함되어 있는지 확인"""
    keywords = ["시", "소설", "이야기", "글쓰기", "창작", "작문"]
    return any(kw in input_dict["input"].lower() for kw in keywords)

# RunnableBranch로 조건부 라우팅
model = ChatOpenAI(model="gpt-4o", temperature=0.7)

branch = RunnableBranch(
    (is_code_question, code_prompt | model),       # 코드 질문 → 코드 전문가
    (is_creative_request, creative_prompt | model), # 창작 요청 → 작가
    general_prompt | model,                         # 기본값 → 일반 비서
)

chain = branch | StrOutputParser()

# 사용 예시
# result = chain.invoke({"input": "파이썬으로 피보나치 함수 만들어줘"})
# → code_prompt로 라우팅됨
```

`RunnableBranch`는 `(조건함수, Runnable)` 튜플의 리스트를 받고, 마지막 인자는 어떤 조건도 맞지 않을 때의 기본(default) Runnable입니다. Python의 `if/elif/else`와 같은 구조죠.

### 개념 3: 프롬프트 합성 — 레고 블록처럼 조립하기

> 💡 **비유**: 레고로 집을 만든다고 생각해 보세요. 지붕, 벽, 창문을 따로 만들어 두면, 나중에 자유롭게 조합해서 다양한 집을 만들 수 있습니다. 프롬프트 합성도 마찬가지예요 — 프롬프트의 각 부분을 독립적인 블록으로 만들고, 필요에 따라 조합합니다.

`ChatPromptTemplate`은 `+` 연산자, `append()`, `extend()` 메서드를 지원해서, 여러 프롬프트 조각을 자유롭게 합성할 수 있습니다.

#### `+` 연산자로 프롬프트 결합

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.messages import AIMessage

# 기본 프롬프트
base_prompt = ChatPromptTemplate.from_messages([
    ("system", "당신은 {role} 전문가입니다."),
])

# Few-shot 예제 추가 (+ 연산자)
with_examples = (
    base_prompt
    + ("human", "파이썬에서 리스트 정렬은?")  # 문자열 → HumanMessagePromptTemplate
    + AIMessage(content="sorted() 함수나 .sort() 메서드를 사용합니다.")
    + ("human", "{question}")  # 실제 질문
)

result = with_examples.invoke({
    "role": "Python",
    "question": "딕셔너리 합치는 방법은?"
})

for msg in result.to_messages():
    print(f"[{msg.type}] {msg.content}")
# 출력:
# [system] 당신은 Python 전문가입니다.
# [human] 파이썬에서 리스트 정렬은?
# [ai] sorted() 함수나 .sort() 메서드를 사용합니다.
# [human] 딕셔너리 합치는 방법은?
```

여기서 중요한 규칙이 있어요:
- **변수가 없는 고정 메시지**는 `AIMessage(...)` 같은 메시지 객체를 직접 사용
- **변수가 있는 템플릿**은 `("human", "{question}")` 같은 튜플 사용
- **문자열만 전달**하면 자동으로 `HumanMessagePromptTemplate`로 추론

#### `append()`와 `extend()`로 동적 조립

```python
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder

# 기본 프롬프트 생성
prompt = ChatPromptTemplate.from_messages([
    ("system", "당신은 도움이 되는 AI입니다."),
])

# 조건에 따라 동적으로 메시지 추가
use_history = True  # 대화 기록 사용 여부

if use_history:
    prompt = prompt + MessagesPlaceholder("chat_history", optional=True)

# 항상 마지막에 사용자 질문 추가
prompt.append(("human", "{question}"))

print(prompt.input_variables)
# 출력: ['question']
# (chat_history는 optional이므로 input_variables에 포함되지 않음)
```

> 🔥 **실무 팁**: 프롬프트 합성은 **조건부 프롬프트 구성**에 특히 유용합니다. 예를 들어, 관리자 모드일 때만 특정 시스템 메시지를 추가하거나, 디버그 모드일 때만 상세한 지시를 포함하는 식으로 활용할 수 있어요.

### 개념 4: 멀티모달 프롬프트 — 이미지와 텍스트를 함께

> 💡 **비유**: 의사에게 진찰을 받을 때, 증상을 말로만 설명하는 것보다 X-ray 사진을 함께 보여주면 진단이 훨씬 정확해지죠? 멀티모달 프롬프트도 마찬가지입니다 — 텍스트만으로 설명하기 어려운 정보를 이미지와 함께 전달하면 LLM의 이해력이 크게 향상됩니다.

GPT-4o, Claude 3.5, Gemini 같은 최신 모델은 텍스트와 이미지를 동시에 처리할 수 있습니다. LangChain에서는 `HumanMessage`의 `content`에 리스트 형태로 텍스트와 이미지를 함께 전달합니다.

#### URL 기반 이미지 전달

```python
from langchain_core.messages import HumanMessage, SystemMessage
from langchain_openai import ChatOpenAI

model = ChatOpenAI(model="gpt-4o")

# content에 텍스트와 이미지 URL을 리스트로 전달
message = HumanMessage(
    content=[
        {"type": "text", "text": "이 이미지에 무엇이 보이나요? 한국어로 설명해주세요."},
        {
            "type": "image_url",
            "image_url": {"url": "https://upload.wikimedia.org/wikipedia/commons/thumb/a/a7/Camponotus_flavomarginatus_ant.jpg/320px-Camponotus_flavomarginatus_ant.jpg"},
        },
    ]
)

# response = model.invoke([message])

```

#### Base64 인코딩 이미지 전달

로컬 파일이나 동적으로 생성된 이미지는 Base64로 인코딩해서 전달합니다.

```python
import base64
from pathlib import Path
from langchain_core.messages import HumanMessage

def encode_image(image_path: str) -> str:
    """이미지 파일을 Base64 문자열로 인코딩"""
    with open(image_path, "rb") as f:
        return base64.b64encode(f.read()).decode("utf-8")
```

#### ChatPromptTemplate에 멀티모달 통합

템플릿 안에서 멀티모달 콘텐츠를 사용하려면 메시지 객체를 직접 구성합니다.

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.messages import SystemMessage

# 멀티모달 프롬프트 템플릿
prompt = ChatPromptTemplate.from_messages([
    SystemMessage(content="당신은 이미지 분석 전문가입니다. 한국어로 답변하세요."),
    (
        "human",
        [
            {"type": "text", "text": "{question}"},
            {
                "type": "image_url",
                "image_url": {"url": "{image_url}"},
            },
        ],
    ),
])

# 변수 바인딩
result = prompt.invoke({
    "question": "이 다이어그램의 구조를 설명해주세요.",
    "image_url": "https://example.com/architecture.png",
})

# model.invoke(result) 로 실행
```

여기서 핵심은 `("human", [...])` 튜플의 두 번째 요소가 문자열이 아니라 **리스트**라는 점이에요. 리스트 안의 각 딕셔너리가 텍스트(`"type": "text"`)이거나 이미지(`"type": "image_url"`)일 수 있습니다.

## 실습: 직접 해보기

아래는 오늘 배운 네 가지 패턴을 모두 활용한 종합 실습입니다 — 대화 기록을 기억하면서, 입력 유형에 따라 적절한 프롬프트를 선택하는 스마트 어시스턴트를 만들어 봅니다.

```python
"""
Session 3.2 실습: 고급 프롬프트 패턴 종합 예제
- MessagesPlaceholder로 대화 기록 관리
- RunnableBranch로 조건부 프롬프트 라우팅
- 프롬프트 합성으로 동적 구성
"""

import os
from dotenv import load_dotenv
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_core.messages import HumanMessage, AIMessage, SystemMessage
from langchain_core.runnables import RunnableBranch, RunnableLambda
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI

load_dotenv()

# ── 1단계: 목적별 프롬프트 블록 준비 ──────────────────

# 공통 시스템 메시지 블록
base_system = "당신은 다재다능한 AI 비서 '코디'입니다. 항상 한국어로 답변합니다."

# 코드 전문가 프롬프트 (대화 기록 포함)
code_prompt = ChatPromptTemplate.from_messages([
    ("system", base_system + " 당신은 특히 프로그래밍에 능숙합니다. 코드 예제를 포함하여 답변하세요."),
    MessagesPlaceholder("chat_history", optional=True),  # 대화 기록 삽입
    ("human", "{input}"),
])

# 일반 대화 프롬프트 (대화 기록 포함)
general_prompt = ChatPromptTemplate.from_messages([
    ("system", base_system + " 친절하고 이해하기 쉽게 설명하세요."),
    MessagesPlaceholder("chat_history", optional=True),
    ("human", "{input}"),
])

# ── 2단계: 조건 판별 함수 ──────────────────────────

CODE_KEYWORDS = {"코드", "프로그래밍", "함수", "버그", "에러", "python", "java",
                 "javascript", "변수", "클래스", "api", "디버깅", "알고리즘"}

def is_code_related(input_dict: dict) -> bool:
    """입력이 코드 관련인지 판별"""
    text = input_dict["input"].lower()
    return any(keyword in text for keyword in CODE_KEYWORDS)

# ── 3단계: 조건부 라우팅 체인 구성 ──────────────────

model = ChatOpenAI(model="gpt-4o", temperature=0.7)

router = RunnableBranch(
    (is_code_related, code_prompt | model | StrOutputParser()),
    general_prompt | model | StrOutputParser(),  # 기본 라우트
)

# ── 4단계: 대화 기록 관리 ──────────────────────────

chat_history: list = []  # 대화 기록 저장소

def chat(user_input: str) -> str:
    """사용자 입력을 받아 응답을 반환하고 대화 기록을 갱신"""
    response = router.invoke({
        "input": user_input,
        "chat_history": chat_history,
    })

    # 대화 기록 갱신
    chat_history.append(HumanMessage(content=user_input))
    chat_history.append(AIMessage(content=response))

    return response

# ── 5단계: 실행 ──────────────────────────────────

if __name__ == "__main__":
    # 첫 번째 대화 — 일반 질문 (general_prompt로 라우팅)
    print("👤 사용자: 오늘 날씨가 좋네요!")
    print(f"🤖 코디: {chat('오늘 날씨가 좋네요!')}")
    print()

    # 두 번째 대화 — 코드 질문 (code_prompt로 라우팅)
    print("👤 사용자: 파이썬으로 피보나치 함수 만들어줘")
    print(f"🤖 코디: {chat('파이썬으로 피보나치 함수 만들어줘')}")
    print()

    # 세 번째 대화 — 이전 맥락 참조 (대화 기록 활용)
    print("👤 사용자: 방금 만든 함수를 메모이제이션으로 최적화해줘")
    print(f"🤖 코디: {chat('방금 만든 함수를 메모이제이션으로 최적화해줘')}")
    print()

    # 대화 기록 확인
    print(f"📝 대화 기록: {len(chat_history)}개의 메시지")
    for msg in chat_history:
        label = "👤" if isinstance(msg, HumanMessage) else "🤖"
        print(f"  {label} [{msg.type}] {msg.content[:50]}...")
```

이 실습을 실행하면:
1. 일반 질문은 `general_prompt`로, 코드 질문은 `code_prompt`로 자동 라우팅됩니다.
2. `MessagesPlaceholder`를 통해 이전 대화가 자동으로 프롬프트에 삽입됩니다.
3. 세 번째 질문에서 "방금 만든 함수"라는 문맥이 대화 기록 덕분에 자연스럽게 이해됩니다.

## 더 깊이 알아보기

### "프롬프트도 프로그래밍이다" — 프롬프트 엔지니어링의 탄생 배경

프롬프트 엔지니어링이라는 개념은 2020년 OpenAI의 GPT-3 논문 *"Language Models are Few-Shot Learners"*에서 본격적으로 주목받았습니다. 이 논문에서 Tom Brown 등의 연구자들은 모델을 재학습(fine-tuning)하지 않고도, **프롬프트에 몇 개의 예제를 넣는 것만으로** 다양한 작업을 수행할 수 있음을 보여줬어요. 이것이 바로 우리가 알고 있는 Few-shot 프롬프팅의 시작이었죠.

LangChain의 창시자 Harrison Chase는 2022년 10월에 LangChain을 처음 공개했는데, 핵심 설계 철학 중 하나가 바로 **"프롬프트를 코드처럼 관리하자"**였습니다. 그가 `PromptTemplate`과 `ChatPromptTemplate`을 설계한 이유는 프롬프트를 하드코딩된 문자열이 아닌, 조합 가능하고 테스트 가능한 소프트웨어 컴포넌트로 다루기 위해서였어요. 오늘 배운 프롬프트 합성과 조건부 라우팅은 바로 이 철학의 연장선입니다.

### PipelinePromptTemplate — 합성의 원조 (그리고 그 이후)

사실 프롬프트 합성을 위한 전용 클래스가 LangChain에 존재했습니다 — `PipelinePromptTemplate`이 그것이죠. 여러 `PromptTemplate`을 파이프라인처럼 연결해서, 한 프롬프트의 출력을 다른 프롬프트의 입력 변수로 넘길 수 있었습니다.

하지만 LangChain 0.3.22 이후로 이 클래스는 **deprecated(사용 중단 권고)** 상태가 되었습니다. 이유는 LCEL의 발전과 함께, 체인 조합만으로 동일한 (더 유연한) 결과를 얻을 수 있게 되었기 때문이에요. 오늘 배운 `+` 연산자와 `append`/`extend`가 사실상 `PipelinePromptTemplate`의 역할을 대체하고 있는 셈입니다.

## 흔한 오해와 팁

> ⚠️ **흔한 오해**: "`MessagesPlaceholder`는 챗봇에서만 쓰인다"고 생각하기 쉽지만, 사실 **어떤 메시지 리스트든** 동적으로 삽입하는 데 사용할 수 있습니다. Few-shot 예제 리스트, 도구 호출 결과 목록, 에이전트의 중간 사고 과정 등 활용 범위가 훨씬 넓어요.

> 💡 **알고 계셨나요?**: 멀티모달 프롬프트의 `image_url`에 전달하는 이미지는 **모델 호출 시 토큰으로 변환**됩니다. 고해상도 이미지 하나가 수백~수천 토큰을 소비할 수 있어요. LangChain 공식 문서에서도 "가능하면 이미지를 압축하고, URL이 접근 가능한 경우 Base64 대신 URL을 사용하라"고 권장합니다.

> 🔥 **실무 팁**: `RunnableBranch`의 조건 함수에서 키워드 매칭 대신 **LLM 기반 분류**를 사용하면 라우팅 정확도가 크게 올라갑니다. 입력을 먼저 가벼운 모델(예: GPT-4o-mini)로 분류한 뒤, 그 결과에 따라 전문 프롬프트로 라우팅하는 2단계 패턴이 실무에서 많이 쓰입니다.

## 핵심 정리

| 개념 | 설명 |
|------|------|
| `MessagesPlaceholder` | 프롬프트 내에 메시지 리스트를 동적으로 삽입하는 자리표시자. `optional=True`로 빈 리스트 허용 |
| 튜플 단축 문법 | `("placeholder", "{변수명}")` — `MessagesPlaceholder`의 간편 표기법 |
| `RunnableBranch` | `(조건함수, Runnable)` 쌍으로 입력을 조건부 라우팅. `if/elif/else`와 유사한 구조 |
| `+` 연산자 | `ChatPromptTemplate` 간 합성. 메시지 객체, 튜플, 문자열을 결합 가능 |
| `append()` / `extend()` | 프롬프트에 메시지를 동적으로 추가. 조건부 프롬프트 구성에 유용 |
| 멀티모달 `content` | `HumanMessage`의 `content`에 `[{"type": "text", ...}, {"type": "image_url", ...}]` 리스트 전달 |
| Base64 이미지 | `data:image/png;base64,{데이터}` 형식으로 로컬 이미지를 인라인 전달 |
