# 출력 파서 기초

> LLM의 자유로운 텍스트 응답을 프로그래밍에서 사용할 수 있는 구조화된 데이터로 변환하는 출력 파서의 기본 개념과 사용법을 학습합니다.

## 개요

이 섹션에서는 LangChain의 출력 파서(Output Parser)가 왜 필요한지, 어떤 원리로 동작하는지를 이해하고, 가장 기본적인 두 가지 파서인 `StrOutputParser`와 `JsonOutputParser`를 직접 사용해봅니다.

**선수 지식**: [Ch3: 프롬프트 엔지니어링과 템플릿](ch03)에서 배운 `ChatPromptTemplate`, `PromptTemplate`, 그리고 [Ch5 예고] LCEL 파이프 연산자(`|`)의 기본 개념
**학습 목표**:
- 출력 파서의 역할과 동작 원리를 설명할 수 있다
- `StrOutputParser`를 사용하여 모델 응답을 문자열로 추출할 수 있다
- `JsonOutputParser`를 사용하여 LLM 응답을 JSON 객체로 파싱할 수 있다
- `format_instructions`를 프롬프트에 주입하여 LLM이 원하는 형식으로 응답하도록 유도할 수 있다

## 왜 알아야 할까?

LLM에게 "사용자 정보를 분석해줘"라고 요청하면, 모델은 자유로운 텍스트로 응답합니다. "이 사용자는 30대 남성으로, 서울에 거주하며..." 이런 식이죠. 사람이 읽기에는 좋지만, 이걸 데이터베이스에 저장하거나 다른 API에 전달하려면 어떻게 해야 할까요?

```python
# LLM의 원시 응답 — AIMessage 객체
AIMessage(content='이 사용자는 30대 남성으로, 서울에 거주합니다.')

# 우리가 원하는 것 — 구조화된 데이터
{"age_group": "30대", "gender": "남성", "city": "서울"}
```

바로 이 간극을 메워주는 것이 **출력 파서(Output Parser)**입니다. 출력 파서는 LLM의 자유분방한 텍스트 응답을 프로그래밍에서 바로 사용할 수 있는 형태로 변환해줍니다. 실제 프로덕션 애플리케이션에서는 LLM 응답을 JSON으로 파싱하여 프론트엔드에 전달하거나, 구조화된 데이터를 다음 체인(Chain)의 입력으로 넘기는 것이 일상적인 작업이거든요.

## 핵심 개념

### 개념 1: 출력 파서의 역할 — 통역사 비유

> 💡 **비유**: 출력 파서는 **동시통역사**와 같습니다. 외국어 연설자(LLM)가 자유롭게 말한 내용을 듣고, 정해진 양식(구조화된 데이터)에 맞게 번역해주는 거죠. 통역사가 연설자에게 "이 부분은 목록 형태로 말씀해주세요"라고 미리 요청할 수 있듯이, 출력 파서도 `format_instructions`를 통해 LLM에게 "이런 형식으로 답변해줘"라고 안내합니다.

LangChain의 출력 파서는 크게 **두 가지 역할**을 수행합니다:

1. **포맷 지시(Format Instructions)**: LLM에게 "이런 형식으로 답해달라"는 안내문을 생성합니다
2. **파싱(Parsing)**: LLM의 텍스트 응답을 실제 Python 데이터 구조로 변환합니다

모든 출력 파서는 `BaseOutputParser`를 상속받으며, 두 가지 핵심 메서드를 제공합니다:

```python
# 모든 출력 파서의 핵심 메서드
parser.get_format_instructions()  # LLM에게 전달할 형식 안내문 반환
parser.parse(text)                # LLM 응답 텍스트를 구조화된 데이터로 변환
```

### 개념 2: StrOutputParser — 가장 단순한 파서

> 💡 **비유**: `StrOutputParser`는 **택배 상자에서 물건만 꺼내주는 사람**입니다. LLM의 응답은 `AIMessage`라는 상자에 담겨 오는데, `StrOutputParser`는 그 상자를 열고 안에 있는 텍스트(content)만 깔끔하게 꺼내줍니다.

`StrOutputParser`는 LangChain에서 가장 간단한 출력 파서입니다. `AIMessage` 객체에서 `content` 문자열만 추출하는 것이 전부인데요, 단순해 보이지만 LCEL 체인에서 매우 자주 사용됩니다.

```python
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate

# 모델과 프롬프트 설정
model = ChatOpenAI(model="gpt-4o", temperature=0.7)
prompt = ChatPromptTemplate.from_template("{topic}에 대해 한 문장으로 설명해주세요.")

# StrOutputParser 없이 — AIMessage 객체가 반환됨
result_without_parser = (prompt | model).invoke({"topic": "인공지능"})
print(type(result_without_parser))
# <class 'langchain_core.messages.ai.AIMessage'>
print(result_without_parser)
# content='인공지능은 인간의 학습 능력을 모방한 컴퓨터 시스템입니다.' ...

# StrOutputParser 사용 — 깔끔한 문자열만 반환됨
parser = StrOutputParser()
result_with_parser = (prompt | model | parser).invoke({"topic": "인공지능"})
print(type(result_with_parser))
# <class 'str'>
print(result_with_parser)
# "인공지능은 인간의 학습 능력을 모방한 컴퓨터 시스템입니다."
```

`StrOutputParser`는 스트리밍도 지원합니다. LCEL 체인에서 `.stream()` 호출 시 텍스트 청크를 그대로 전달해주죠:

```python
# 스트리밍 사용 예시
chain = prompt | model | StrOutputParser()

for chunk in chain.stream({"topic": "양자 컴퓨팅"}):
    print(chunk, end="", flush=True)
# 양자 컴퓨팅은 양자역학의 원리를 활용하여...
```

### 개념 3: JsonOutputParser — 구조화된 출력의 시작

> 💡 **비유**: `JsonOutputParser`는 **인터뷰 속기사**와 같습니다. 인터뷰 전에 "이름, 나이, 직업 순서로 답변해주세요"라는 양식(format_instructions)을 건네고, 답변이 끝나면 그 내용을 정해진 양식에 맞게 정리(parse)해줍니다.

`JsonOutputParser`는 LLM의 텍스트 응답을 Python 딕셔너리(JSON 객체)로 변환합니다. 함수 호출(function calling)을 사용하지 않으면서도 구조화된 데이터를 얻을 수 있는 가장 신뢰성 높은 방법이에요.

```python
from langchain_core.output_parsers import JsonOutputParser
from langchain_core.prompts import PromptTemplate

# 기본 JsonOutputParser — 스키마 없이 사용
parser = JsonOutputParser()

prompt = PromptTemplate(
    template="다음 과일의 정보를 JSON 형식으로 알려주세요.\n{format_instructions}\n과일: {fruit}",
    input_variables=["fruit"],
    partial_variables={"format_instructions": parser.get_format_instructions()},
)

chain = prompt | model | parser
result = chain.invoke({"fruit": "사과"})
print(result)
# {'name': '사과', 'color': '빨간색', 'taste': '달콤새콤', 'origin': '중앙아시아'}
print(type(result))
# <class 'dict'>
```

### 개념 4: format_instructions의 동작 원리

`format_instructions`는 출력 파서가 자동으로 생성하는 **형식 안내문**입니다. 이 안내문이 프롬프트에 삽입되어 LLM이 파서가 이해할 수 있는 형식으로 응답하도록 유도합니다.

실제로 `JsonOutputParser`의 `get_format_instructions()`가 어떤 텍스트를 생성하는지 확인해볼까요?

```python
from langchain_core.output_parsers import JsonOutputParser

parser = JsonOutputParser()
print(parser.get_format_instructions())
```

출력 결과:
```
Return a JSON object.
```

Pydantic 모델을 지정하면 더 구체적인 안내문이 생성됩니다:

```python
from pydantic import BaseModel, Field
from langchain_core.output_parsers import JsonOutputParser

# Pydantic 모델로 스키마 정의
class BookInfo(BaseModel):
    title: str = Field(description="책 제목")
    author: str = Field(description="저자 이름")
    year: int = Field(description="출판 연도")
    genre: str = Field(description="장르")

parser = JsonOutputParser(pydantic_object=BookInfo)
print(parser.get_format_instructions())
```

출력 결과:
```
The output should be formatted as a JSON instance that conforms to the JSON schema below.

As an example, for the schema {"properties": {"foo": {"title": "Foo", "description": "a list of strings", "type": "array", "items": {"type": "string"}}}, "required": ["foo"]}, the object {"foo": ["bar", "baz"]} is a well-formatted instance of the schema. The object {"properties": {"foo": ["bar", "baz"]}} is not well-formatted.

Here is the output schema:
{"properties": {"title": {"description": "책 제목", "title": "Title", "type": "string"}, "author": {"description": "저자 이름", "title": "Author", "type": "string"}, "year": {"description": "출판 연도", "title": "Year", "type": "integer"}, "genre": {"description": "장르", "title": "Genre", "type": "string"}}, "required": ["title", "author", "year", "genre"]}
```

이 안내문이 프롬프트에 포함되면, LLM은 지정된 JSON 스키마에 맞춰 응답을 생성하게 됩니다. `partial_variables`를 사용하면 이 과정을 깔끔하게 처리할 수 있어요:

```python
prompt = PromptTemplate(
    template="다음 책의 정보를 알려주세요.\n{format_instructions}\n책: {book_name}",
    input_variables=["book_name"],
    # format_instructions를 미리 채워넣기
    partial_variables={"format_instructions": parser.get_format_instructions()},
)

chain = prompt | model | parser
result = chain.invoke({"book_name": "해리 포터와 마법사의 돌"})
print(result)
# {'title': '해리 포터와 마법사의 돌', 'author': 'J.K. 롤링', 'year': 1997, 'genre': '판타지'}
```

### 개념 5: JsonOutputParser의 스트리밍 지원

`JsonOutputParser`의 강력한 특징 중 하나는 **부분 JSON 스트리밍**을 지원한다는 점입니다. LLM이 토큰을 생성하는 도중에도 지금까지 파싱된 부분 객체를 계속 반환해줍니다:

```python
chain = prompt | model | parser

# 스트리밍 시 부분 JSON 객체가 점진적으로 반환됨
for partial in chain.stream({"book_name": "1984"}):
    print(partial)

# {'title': ''}
# {'title': '1984'}
# {'title': '1984', 'author': ''}
# {'title': '1984', 'author': '조지 오웰'}
# {'title': '1984', 'author': '조지 오웰', 'year': 1949}
# {'title': '1984', 'author': '조지 오웰', 'year': 1949, 'genre': '디스토피아'}
```

이 기능은 실시간 UI 업데이트가 필요한 애플리케이션에서 매우 유용합니다. 사용자에게 "로딩 중..."을 보여주는 대신, 데이터가 채워지는 과정을 실시간으로 보여줄 수 있거든요.

## 실습: 직접 해보기

영화 정보를 구조화된 JSON으로 추출하는 완전한 예제를 만들어봅시다.

```python
"""
영화 정보 추출기 — JsonOutputParser 실습
LLM에게 영화 이름을 주면 구조화된 정보를 반환합니다.
"""
import os
from dotenv import load_dotenv
from pydantic import BaseModel, Field
from langchain_core.output_parsers import JsonOutputParser, StrOutputParser
from langchain_core.prompts import ChatPromptTemplate, PromptTemplate
from langchain_openai import ChatOpenAI

# 환경 변수 로드
load_dotenv()

# 모델 설정
MODEL_NAME = "gpt-4o"
TEMPERATURE = 0.3  # 정확한 정보 추출을 위해 낮은 temperature

model = ChatOpenAI(model=MODEL_NAME, temperature=TEMPERATURE)


# ===== 1단계: StrOutputParser로 간단한 요약 =====
print("=" * 50)
print("1단계: StrOutputParser — 텍스트 요약")
print("=" * 50)

str_prompt = ChatPromptTemplate.from_template(
    "영화 '{movie}'를 한 문장으로 소개해주세요."
)
str_chain = str_prompt | model | StrOutputParser()

summary = str_chain.invoke({"movie": "인터스텔라"})
print(f"요약: {summary}")
print(f"타입: {type(summary)}")  # <class 'str'>


# ===== 2단계: JsonOutputParser로 구조화된 정보 추출 =====
print("\n" + "=" * 50)
print("2단계: JsonOutputParser — 구조화된 정보")
print("=" * 50)

# Pydantic 모델로 출력 스키마 정의
class MovieInfo(BaseModel):
    title: str = Field(description="영화 제목 (원제)")
    korean_title: str = Field(description="한국어 제목")
    director: str = Field(description="감독 이름")
    year: int = Field(description="개봉 연도")
    genre: list[str] = Field(description="장르 목록")
    rating: float = Field(description="IMDb 평점 (10점 만점)")
    plot_summary: str = Field(description="줄거리 요약 (2-3문장)")

# JsonOutputParser에 Pydantic 모델 전달
json_parser = JsonOutputParser(pydantic_object=MovieInfo)

json_prompt = PromptTemplate(
    template=(
        "다음 영화의 상세 정보를 알려주세요.\n"
        "{format_instructions}\n"
        "영화: {movie}"
    ),
    input_variables=["movie"],
    partial_variables={
        "format_instructions": json_parser.get_format_instructions()
    },
)

json_chain = json_prompt | model | json_parser

# 영화 정보 추출
movie_data = json_chain.invoke({"movie": "인터스텔라"})
print(f"타입: {type(movie_data)}")  # <class 'dict'>
print(f"제목: {movie_data['korean_title']}")
print(f"감독: {movie_data['director']}")
print(f"연도: {movie_data['year']}")
print(f"장르: {', '.join(movie_data['genre'])}")
print(f"평점: {movie_data['rating']}")
print(f"줄거리: {movie_data['plot_summary']}")


# ===== 3단계: 여러 영화를 한번에 처리 =====
print("\n" + "=" * 50)
print("3단계: 배치 처리 — 여러 영화 동시 분석")
print("=" * 50)

movies = ["기생충", "올드보이", "부산행"]
results = json_chain.batch([{"movie": m} for m in movies])

for movie_info in results:
    print(f"\n🎬 {movie_info['korean_title']} ({movie_info['year']})")
    print(f"   감독: {movie_info['director']}")
    print(f"   장르: {', '.join(movie_info['genre'])}")
    print(f"   평점: {movie_info['rating']}")


# ===== 4단계: 스트리밍으로 실시간 파싱 =====
print("\n" + "=" * 50)
print("4단계: 스트리밍 — 실시간 JSON 파싱")
print("=" * 50)

print("스트리밍 중...")
for partial_result in json_chain.stream({"movie": "매트릭스"}):
    # 현재까지 파싱된 키 목록 출력
    keys = list(partial_result.keys())
    print(f"  파싱된 키: {keys}")

print(f"\n최종 결과: {partial_result['korean_title']} - {partial_result['director']}")
```

## 더 깊이 알아보기

### 출력 파서의 탄생 배경

출력 파서는 LangChain 초기부터 존재했던 핵심 컴포넌트입니다. 2022년 말 LangChain이 처음 등장했을 때, GPT-3.5조차 함수 호출(function calling) 기능을 지원하지 않았어요. 당시 LLM에게 구조화된 데이터를 얻으려면 프롬프트에 "JSON 형식으로 답해줘"라고 써야 했는데, 이것만으로는 불안정했습니다.

LangChain의 창시자 **해리슨 체이스(Harrison Chase)**는 이 문제를 **"포맷 지시 + 파싱"**이라는 두 단계로 해결했습니다. 프롬프트에 정확한 형식 안내를 자동으로 삽입하고, 응답을 자동으로 파싱하는 구조를 만든 거죠. 이 패턴은 너무 유용해서 이후 다른 LLM 프레임워크에서도 비슷한 방식을 채택했습니다.

흥미로운 점은, 2023년 중반 OpenAI가 함수 호출 기능을 도입한 이후 출력 파서의 역할이 변화했다는 것입니다. 이제는 `with_structured_output()`이라는 더 간편한 방법이 생겼지만, 출력 파서는 여전히 함수 호출을 지원하지 않는 모델이나 세밀한 커스터마이징이 필요한 경우에 핵심적인 역할을 합니다. 이에 대해서는 이 챕터의 이후 섹션에서 자세히 다루겠습니다.

### 출력 파서 vs with_structured_output

LangChain 공식 문서에서는 최신 모델의 경우 `with_structured_output()` 사용을 권장합니다. 하지만 출력 파서를 먼저 배우는 것이 중요한 이유가 있어요:

- **원리 이해**: `with_structured_output()`이 내부적으로 하는 일을 이해할 수 있습니다
- **호환성**: 모든 LLM이 함수 호출을 지원하지는 않습니다
- **유연성**: 커스텀 파싱 로직이 필요할 때 출력 파서가 필수입니다
- **스트리밍**: `JsonOutputParser`의 부분 JSON 스트리밍은 고유한 장점입니다

## 흔한 오해와 팁

> ⚠️ **흔한 오해**: "StrOutputParser는 불필요하다" — 초보자들이 자주 하는 오해입니다. `model.invoke()`로도 응답을 받을 수 있으니 파서가 왜 필요한지 의문이 드실 수 있어요. 하지만 LCEL 체인에서 `StrOutputParser` 없이 다음 컴포넌트로 `AIMessage` 객체가 전달되면, 문자열을 기대하는 후속 컴포넌트에서 오류가 발생합니다. 특히 `RunnablePassthrough`나 다른 체인과 조합할 때 `StrOutputParser`는 거의 필수입니다.

> 💡 **알고 계셨나요?**: `JsonOutputParser`는 LLM 응답에 JSON 외의 텍스트가 섞여 있어도 JSON 부분만 추출하려고 시도합니다. 예를 들어 LLM이 "다음은 요청하신 JSON입니다: `{...}`"라고 응답해도, 마크다운 코드 블록 안의 JSON을 찾아서 파싱해줍니다. 하지만 이 동작에 의존하기보다는, `format_instructions`를 활용하여 처음부터 깔끔한 JSON만 출력하도록 유도하는 것이 좋습니다.

> 🔥 **실무 팁**: `JsonOutputParser`에 Pydantic 모델을 지정하면 `format_instructions`가 JSON Schema를 포함하게 되어 파싱 성공률이 크게 올라갑니다. 하지만 주의할 점이 있어요 — `JsonOutputParser`는 실제로 Pydantic 검증(validation)을 수행하지 않고 단순히 JSON으로만 파싱합니다. 타입 검증까지 원한다면 다음 섹션에서 배울 `PydanticOutputParser`를 사용하세요.

## 핵심 정리

| 개념 | 설명 |
|------|------|
| 출력 파서(Output Parser) | LLM의 텍스트 응답을 구조화된 데이터로 변환하는 컴포넌트 |
| `get_format_instructions()` | LLM에게 전달할 형식 안내문을 생성하는 메서드 |
| `parse(text)` | LLM 응답 텍스트를 파싱하여 원하는 데이터 구조로 변환하는 메서드 |
| `StrOutputParser` | `AIMessage`에서 텍스트 content만 추출하는 가장 단순한 파서 |
| `JsonOutputParser` | LLM 응답을 Python 딕셔너리(JSON)로 변환하는 파서 |
| `partial_variables` | 프롬프트 템플릿에 `format_instructions`를 미리 주입하는 방법 |
| Pydantic 스키마 연동 | `JsonOutputParser(pydantic_object=Model)`로 구체적인 형식 안내 생성 |
| 부분 JSON 스트리밍 | `JsonOutputParser`가 스트리밍 시 점진적으로 JSON 객체를 반환하는 기능 |

## 다음 섹션 미리보기

이번 섹션에서 `JsonOutputParser`가 Pydantic 모델의 스키마만 참고하고 실제 검증은 하지 않는다는 것을 배웠죠? 다음 섹션 **[4.2: PydanticOutputParser와 타입 안전 출력]**에서는 Pydantic의 타입 검증까지 활용하여, LLM 응답이 정의한 스키마에 정확히 부합하는지까지 보장하는 방법을 학습합니다. 타입 안전성이 중요한 프로덕션 환경에서 필수적인 내용이니 기대해주세요.
